   ALTER TABLE "Session" ENABLE ROW LEVEL SECURITY;

   - `CREATE OR REPLACE FUNCTION public.is_trainer()` (idempotent)
   - Returns `boolean`, `LANGUAGE sql`, `SECURITY DEFINER`, `STABLE`

3. **Policies for Associate** (per D-04, D-05):
   - SELECT: `"authUserId" = auth.uid()::text OR public.is_trainer()` (cast uuid to text per Research Pitfall 1)
   - Write (FOR ALL): `public.is_trainer()` in both USING and WITH CHECK

4. **Policies for Session** (per D-04, D-05):
   - SELECT: `public.is_trainer() OR EXISTS (SELECT 1 FROM "Associate" a WHERE a.id = "Session"."associateId" AND a."authUserId" = auth.uid()::text)`
   - Write (FOR ALL): `public.is_trainer()`

5. **Policies for GapScore** (same pattern as Session):
   - SELECT: `public.is_trainer() OR EXISTS (SELECT 1 FROM "Associate" a WHERE a.id = "GapScore"."associateId" AND a."authUserId" = auth.uid()::text)`
   - Write (FOR ALL): `public.is_trainer()`

6. **Policies for Cohort** (trainer-only per D-04):
   - FOR ALL: `public.is_trainer()` in USING and WITH CHECK

7. **Policies for CurriculumWeek** (trainer-only per D-04):
   - FOR ALL: `public.is_trainer()` in USING and WITH CHECK

Each policy uses `DROP POLICY IF EXISTS` before `CREATE POLICY` for idempotency (per Research "Don't Hand-Roll" section).

Do NOT use `FORCE ROW LEVEL SECURITY` on any table (per Research Pitfall 4 -- would break Prisma service-role writes).

Policy naming convention: `{table_lowercase}_{select|write}` (e.g., `associate_select`, `session_write`).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| supabase-js (anon key) -> Postgres | Direct DB reads via client SDK bypass app-layer auth; RLS is the only guard |
| Prisma (service-role) -> Postgres | Service-role bypasses RLS; app-layer identity filtering is the guard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-01 | Elevation of Privilege | Direct supabase-js query on Session | mitigate | RLS policy `session_select` requires owner match via Associate FK or trainer role |
| T-20-02 | Elevation of Privilege | Direct supabase-js query on GapScore | mitigate | RLS policy `gapscore_select` requires owner match via Associate FK or trainer role |
| T-20-03 | Elevation of Privilege | Direct supabase-js query on Associate | mitigate | RLS policy `associate_select` requires authUserId match or trainer role |
| T-20-04 | Denial of Service | RLS enabled without policies -> all reads return empty | mitigate | Policies created in same atomic migration as ENABLE RLS; rollback on failure |
| T-20-05 | Spoofing | is_trainer() returns wrong result | mitigate | Uses auth.jwt() user_metadata.role set server-side by P18; cannot be tampered by client |
</threat_model>
