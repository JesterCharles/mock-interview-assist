### Human Verification Required

**1. Avatar menu Profile link visual position**

**Test:** Sign in as any authenticated user, click the avatar button in the top-right corner of the trainer dashboard or associate shell.
**Expected:** Dropdown renders email at top, role badge below it, then separator, then "Profile" link, then separator, then "Sign Out" in red.
**Why human:** Radix DropdownMenu portal rendering order requires browser verification. Static source order is correct but rendering depends on Radix internals.

**2. Learning tab readiness data for associate**

**Test:** Log in as an associate who has completed at least one interview session. Navigate to /profile. Click the Learning tab.
**Expected:** Readiness summary card displays status badge (color-coded), score percentage, focus area text, and session count. Below, an editable textarea for learning goals with a "Save goals" button.
**Why human:** Real readiness data requires live DB state with completed sessions and computed gap scores.

**3. First-login detection end-to-end**

**Test:** Invite a new associate via trainer bulk-invite. Use the magic link from their email to log in for the first time (before setting a password).
**Expected:** Exchange route detects no `Profile.passwordSetAt` and no `user_metadata.password_set`, redirects to `/auth/set-password`. After submitting a password, both `user_metadata.password_set` and `Profile.passwordSetAt` are written. Subsequent logins bypass the set-password redirect.
**Why human:** Requires live Supabase auth flow + DB write sequence; cannot be traced programmatically.
