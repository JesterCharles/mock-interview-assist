    2. Create `loadtest/scripts/fetch-cloud-run-metrics.sh`:
       - `#!/usr/bin/env bash`, `set -euo pipefail`.
       - Reads `GCP_PROJECT`, `SERVICE_NAME` from env.
       - Uses `gcloud monitoring time-series list --project="$GCP_PROJECT" --filter='metric.type="run.googleapis.com/container/cpu/utilizations" AND resource.label.service_name="'"$SERVICE_NAME"'"' --format=json --window=15m` (or equivalent `monitoring read` command). Parse with `jq` to compute:
         - `peak_cpu_pct` (max sample across window × 100)
         - `peak_mem_pct` (same, for `container/memory/utilizations`)
         - `total_vcpu_seconds` (integrate CPU utilization × instance-seconds — approximation: sum(cpu_samples) × sample_interval × cpu_allocation_count)
         - `total_gb_seconds` (same for memory × 0.5 GB per instance)
       - Outputs a single JSON blob `{ peak_cpu_pct, peak_mem_pct, total_vcpu_seconds, total_gb_seconds, source: "gcloud monitoring", window_end: "<ISO>" }` to stdout.
       - If any `gcloud` call fails (non-zero exit), emit a JSON error blob `{ error: "<reason>", source: "gcloud monitoring" }` and exit 0 (soft-fail so the workflow still publishes artifacts and Task 2 triages manually).

    3. Create `loadtest/scripts/fetch-supabase-query-count.sh`:
       - `#!/usr/bin/env bash`, `set -euo pipefail`.
       - Uses `gcloud logging read 'resource.type="cloud_run_revision" AND resource.labels.service_name="'"$SERVICE_NAME"'" AND jsonPayload.headers."x-session-id" != ""' --project="$GCP_PROJECT" --freshness=15m --format=json --limit=5000`.
       - Parses with `jq` to count log entries grouped by `jsonPayload.headers."x-session-id"`, emits p50 + p95 of per-session query counts.
       - Output JSON: `{ sessions_observed: N, queries_per_session: { p50, p95, max }, source: "cloud logging", window: "15m" }` to stdout.
       - Soft-fail policy identical to fetch-cloud-run-metrics.sh.
       - **Caveat embedded in the script as a comment**: if the app does not yet instrument Prisma queries with the X-Session-ID header (it doesn't at Phase 49 time), this script will emit `queries_per_session: null` and a note. Plan 02 accepts "instrumentation pending" for the p50/p95 cells in the baseline doc — explicit `TBD — instrumented in v1.6 observability polish`. This is fine per D-08 being a "best-effort" measure, not a hard LOAD-03 blocker.
