<objective>
Add a nullable `prevWeightedScore Float?` column to the GapScore model and extend `saveGapScores` to capture each row's prior weighted score during the session-completion upsert.

Purpose: Per D-07–D-14, real per-skill historical snapshots must replace the synthetic "0.85 * current" calculation that currently drives the SkillRadar "Before" polygon. The snapshot write is inline in `gapPersistence.saveGapScores` — read the existing row's `weightedScore` once, write it into `prevWeightedScore`, then overwrite `weightedScore` with the newly computed value. First upserts leave `prevWeightedScore` null. No backfill of existing rows.
Output: Prisma schema + migration adding the column, updated persistence code with a pre-fetch map and extended upsert, and a unit test proving the three-state lifecycle (null → value → updated) + cleanup integrity.
</objective>

Rewrite `saveGapScores` in `src/lib/gapPersistence.ts` so the existing-row `findMany` used for cleanup ALSO powers the prior-value lookup in a single round-trip. Collapse the two existing `findMany({ where: { associateId } })` calls — one for upsert, one for cleanup — into a single pre-fetch that returns `id`, `skill`, `topic`, `weightedScore`. Build a `Map<string, number>` keyed `"${skill}::${topic}"` → prior `weightedScore`. Drive upserts off that map.

Replace the body of `saveGapScores` from step 4 onward with:

```typescript
  // 3. Compute gap scores
  const gapScores = computeGapScores(sessions);

  if (gapScores.length === 0) return;

  // 4. Pre-fetch existing rows once — supplies both prior-value map AND cleanup ids.
  // D-13: single SELECT per save, no extra round-trip on top of today's cleanup query.
  const existingScores = await prisma.gapScore.findMany({
    where: { associateId },
    select: { id: true, skill: true, topic: true, weightedScore: true },
  });

  const priorByKey = new Map<string, number>();
  for (const row of existingScores) {
    priorByKey.set(`${row.skill}::${row.topic}`, row.weightedScore);
  }

  // 5. Upsert each gap score — capture prior weightedScore into prevWeightedScore.
  // D-08: "before = score as of the previous session". D-09: null on first-ever upsert.
  await Promise.all(
    gapScores.map((input) => {
      const key = `${input.skill}::${input.topic}`;
      const prior = priorByKey.has(key) ? priorByKey.get(key)! : null;
      return prisma.gapScore.upsert({
        where: {
          associateId_skill_topic: {
            associateId,
            skill: input.skill,
            topic: input.topic,
          },
        },
        update: {
          weightedScore: input.weightedScore,
          prevWeightedScore: prior,
          sessionCount: input.sessionCount,
        },
        create: {
          associateId,
          skill: input.skill,
          topic: input.topic,
          weightedScore: input.weightedScore,
          prevWeightedScore: null,
          sessionCount: input.sessionCount,
        },
      });
    }),
  );

  // 6. Delete stale GapScore records not in current computation (reuses existingScores)
  const currentKeys = new Set(
    gapScores.map((g) => `${g.skill}::${g.topic}`),
  );

  const staleIds = existingScores
    .filter((s) => !currentKeys.has(`${s.skill}::${s.topic}`))
    .map((s) => s.id);

  if (staleIds.length > 0) {
    await prisma.gapScore.deleteMany({
      where: { id: { in: staleIds } },
    });
  }
}
```

- The OLD code had two `findMany` calls (one implicit via upsert per-row reads + one explicit for cleanup). This version has ONE `findMany` up front and drives both upsert priors and cleanup off of it. Upsert count is unchanged.
- Do NOT alter `getGapScores` — that function is unchanged (it only reads, and callers that need `prevWeightedScore` hit Prisma directly in other layers).
- Preserve the existing doc comment at top of file; add a one-line note that step 4 supplies prior-value lookup:

vi.mock('@/lib/prisma', () => ({

describe('saveGapScores prevWeightedScore lifecycle', () => {

        technicalFeedback: null, softSkillFeedback: null,

  });

  it('captures prior weightedScore into prevWeightedScore on subsequent upsert', async () => {
    (prisma.gapScore.findMany as ReturnType<typeof vi.fn>).mockResolvedValue([

      { skill: 'React', topic: '', weightedScore: 0.72, sessionCount: 2 },

    const call = (prisma.gapScore.upsert as ReturnType<typeof vi.fn>).mock.calls[0][0];
    expect(call.update.prevWeightedScore).toBe(0.60);
    expect(call.update.weightedScore).toBe(0.72);

  it('updates prevWeightedScore to the value that was current before this save', async () => {

    (computeGapScores as ReturnType<typeof vi.fn>).mockReturnValue([
      { skill: 'React', topic: '', weightedScore: 0.81, sessionCount: 3 },

    await saveGapScores(1);

    const calls = (prisma.gapScore.upsert as ReturnType<typeof vi.fn>).mock.calls.map((c) => c[0]);

    expect(byKey['0.72']).toBe(0.60);
    // SQL skill-level: prior 0.40 → now 0.45
    expect(byKey['0.45']).toBe(0.40);

    expect(byKey['0.66']).toBe(0.55);
