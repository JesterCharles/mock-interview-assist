## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap in KPI cards, badge internal padding, sparkline cell padding |
| sm | 8px | KPI card internal vertical padding, table cell vertical padding, trend tag gap |
| md | 16px | KPI card internal horizontal padding, content section padding, table column gap |
| lg | 24px | KPI strip bottom margin, section heading bottom margin, chart container padding |
| xl | 32px | Gap between KPI strip and roster table, gap between Cohort Trends card and roster table |
| 2xl | 48px | Page-level section breaks |
| 3xl | 64px | Not used in this phase |

Exceptions:
- KPI number text is 28px Clash Display — meets visual scale from DESIGN.md §Typography without changing spacing
- Sparkline cell fixed dimensions: 56px wide × 20px tall (no padding override — Recharts ResponsiveContainer uses explicit pixel dimensions per RESEARCH.md Pitfall 5)
- Roster table row height: 48px minimum (comfortable density per DESIGN.md §Layout)

Source: DESIGN.md §Spacing, 21-UI-SPEC.md §Spacing Scale, CONTEXT.md D-04, D-05

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| KPI value / readiness score | Clash Display | 28px | 600 | 1.2 | KPI card numbers (Avg Readiness %, Mocks count, At-Risk count, AI-Trainer Variance delta) |
| Section heading | Clash Display | 28px | 500 | 1.2 | Page titles: "Gap Analysis", "Calibration", section heading above Cohort Trends chart |
| Card title | Clash Display | 22px | 500 | 1.2 | KPI card label above the number (e.g. "Avg Readiness") — wait, see note |
| KPI label | DM Sans | 12px | 500 | 1.4 | KPI card label below the number (e.g. "avg readiness"), trend word, top-gap tag, last-mock timestamp |
| Table content | DM Sans | 14px | 400 | 1.4 | Roster table cells: associate name, skill/topic columns in Gap Analysis, override frequency |
| Body / UI | DM Sans | 16px | 400 | 1.5 | Empty state body, error state body, chart axis labels |
| Metadata | DM Sans | 12px | 500 | 1.4 | Gap Analysis column headers, Calibration section labels, table sort indicators |

Typography hierarchy correction per CONTEXT.md D-04:
- KPI **numbers** (the metric): 28px Clash Display 600
- KPI **labels** (descriptor text below number): 12px DM Sans (per D-04: "labels are 12px DM Sans")

Weights in use: **400** (table body, body text) and **500/600** (Clash Display headings at 500, KPI numbers at 600, DM Sans labels at 500). No weight 700.

Source: DESIGN.md §Typography, CONTEXT.md D-04, 21-UI-SPEC.md §Typography

## Color

| Role | Token | Hex (light) | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--bg` | `#F5F0E8` | Page background, roster table background |
| Secondary (30%) | `--surface-muted` | `#F0EBE2` | KPI card background, Cohort Trends card background, Calibration section backgrounds |
| Accent (10%) | `--accent` | `#C85A2E` | Sparkline line color, active sort column indicator, primary drill-through link color |
| Destructive | `--danger` | `#B83B2E` | At-Risk count KPI card value (when count > 0), negative delta bars in Calibration histogram |

Accent reserved for:
1. Sparkline line stroke (`stroke="var(--accent)"`) in roster rows
2. Cohort Trends line chart stroke
3. Active column sort indicator in Gap Analysis table (sort arrow in `--accent`)
4. Drill-through row link text in Gap Analysis (hover underline)

NOT used for: KPI card borders, table headers, chart axis labels, positive delta bars

Readiness signal colors (from DESIGN.md §Readiness Signal Pattern):
- `--success` (`#2D6A4F`): trend word "improving", positive delta bars in Calibration histogram, ready-state readiness value
- `--warning` (`#B7791F`): trend word "steady", near-threshold values
- `--danger` (`#B83B2E`): trend word "declining", at-risk values, negative delta bars

Chart-specific colors:
- Calibration delta histogram bars: `--success` for positive deltas (+1, +2, +3), `--muted` for zero, `--danger` for negative (-1, -2, -3)
- Cohort Trends line: `--accent`
- Sparklines: `--accent`

KPI card surfaces: `background: var(--surface-muted)`, `border: 1px solid var(--border)`, `border-radius: 8px` (lg per DESIGN.md §Layout)

Source: DESIGN.md §Color, CONTEXT.md D-04 "DESIGN.md surface tokens", RESEARCH.md discretion items

## Component Inventory

### KpiStrip.tsx
- Layout: CSS Grid, `grid-cols-2` on mobile, `grid-cols-4` on desktop (`md:grid-cols-4`)
- Gap: 16px (md) between cards
- Position: above the roster table, below the page heading on `/trainer`
- Card surface: `background: var(--surface-muted)`, `border: 1px solid var(--border)`, `border-radius: 8px`, `padding: 16px`
- KPI number: 28px Clash Display 600, `color: var(--ink)` (At-Risk count uses `--danger` when > 0)
- KPI label: 12px DM Sans 500, `color: var(--muted)`, displayed below the number
- At-Risk card extra: "Top Gap" label shown as a 12px DM Sans tag below the count in `--muted` text
- AI-Trainer Variance: display as decimal delta (e.g. "+0.4" or "−0.3"), `--success` if positive, `--danger` if negative, `--muted` if zero/null
- Loading state: skeleton bars — `--surface-muted` background blocks at the number height, 150ms fade-in when data loads
- Empty/null state: display "—" in `--muted` when query returns null

Source: CONTEXT.md D-01, D-02, D-04; DESIGN.md §Typography, §Color, §Layout "KPI strip"

### RosterSparkline.tsx
- Dimensions: `<ResponsiveContainer width={56} height={20}>` — explicit pixels, no `width="100%"`
- Chart: `<LineChart>` with `margin={{ top: 2, right: 2, bottom: 2, left: 2 }}`
- Line: `stroke="var(--accent)"`, `strokeWidth={1.5}`, `dot={false}`, `isAnimationActive={false}`
- No axes, no grid, no tooltip, no legend
- Data shape: `Array<{ score: number }>` — last 6 sessions in chronological order (oldest first)
- Single-point edge case: render a horizontal line at that score value
- Zero-session edge case: render nothing (empty cell, 56px wide spacer)
- Table cell: `vertical-align: middle`

Source: CONTEXT.md D-05, D-06; RESEARCH.md Pattern 4

### RosterRowEnrichment (inline in roster table row)
Three new columns added to existing roster table:
1. **Sparkline**: 56px fixed width cell, `RosterSparkline` component
2. **Trend**: trend word + relative timestamp — "improving · 2d ago" — 12px DM Sans 500
   - Color: `--success` for "improving", `--warning` for "steady", `--danger` for "declining"
   - New associates with < 3 sessions: display "new" in `--muted`
   - Timestamp: relative format "today" / "1d ago" / "Nd ago" using inline helper (no library)
3. **Top Gap**: skill name tag — 12px DM Sans 500, `background: var(--surface-muted)`, `border: 1px solid var(--border)`, `border-radius: 4px`, `padding: 2px 6px`

Source: CONTEXT.md D-07, D-08; RESEARCH.md Pattern 5, Open Questions #2

### CohortTrends.tsx
- Card surface: `background: var(--surface-muted)`, `border: 1px solid var(--border)`, `border-radius: 8px`, `padding: 16px 16px 8px`
- Heading: 16px DM Sans 500 `--ink` ("Cohort Readiness — Last 12 Weeks")
- Position: above the KPI strip on `/trainer` (Cohort Trends → KPI Strip → Roster Table)
- Chart: `<LineChart>` with `<ResponsiveContainer width="100%" height={120}>`
- X-axis: week labels "W1"–"W12", 11px DM Sans 500, `--muted`, tick line hidden
- Y-axis: 0–100 domain, hidden axis line, 4 tick values (0, 25, 50, 75, 100), 11px DM Sans 500, `--muted`
- Grid: horizontal dotted lines at tick values, `stroke: var(--border)`, `strokeDasharray="2 4"`
- Line: `stroke="var(--accent)"`, `strokeWidth={2}`, `dot={false}`, `isAnimationActive={false}`
- Tooltip: single-value tooltip showing "W7 · 68% avg", `background: var(--surface)`, `border: 1px solid var(--border)`, `border-radius: 6px`, 12px DM Sans
- Empty state (< 2 weeks of data): show "Not enough data yet — check back after more mocks." in 14px DM Sans `--muted`, centered within the chart area

Source: CONTEXT.md D-20, D-21, D-22

### GapAnalysisTable (Gap Analysis page)
- Page layout: full-width within dashboard content area, no max-width constraint (data-dense table)
- Heading: 28px Clash Display 500 "Gap Analysis"
- Subheading: 14px DM Sans 400 `--muted` "Skill gaps across your cohort · sorted by associates affected"
- Table: 4 columns — Skill, Topic, Associates Affected, Avg Gap Score
- Column widths: Skill (20%), Topic (40%), Associates Affected (20%), Avg Gap Score (20%)
- Header row: 12px DM Sans 500 uppercase `--muted`, 0.04em tracking, `border-bottom: 1px solid var(--border)`
- Body rows: 14px DM Sans 400 `--ink`, row height 48px, `border-bottom: 1px solid var(--border-subtle)`
- Row hover: `background: var(--highlight)` (150ms ease-out)
- Sort: default "Associates Affected DESC" — active column header shows `--accent` sort arrow; clicking header toggles sort direction
- Avg Gap Score: render as 0–100 value (not percentage) with a subtle inline bar — `--danger` fill for score < 50, `--warning` for 50-74, `--success` for 75+; bar max-width 60px at 100
- Drill-through: entire row is clickable, navigates to `/trainer/gap-analysis/[skill]?topic=[topic]&cohort=[id]`; row cursor: pointer; no separate "View" button
- Empty state: see Copywriting Contract
- Loading skeleton: 8 skeleton rows, each with a gray block at label positions

Source: CONTEXT.md D-09, D-10, D-11, D-12

### GapDrillThrough (Gap Analysis drill-through page `/trainer/gap-analysis/[skill]`)
- Back navigation: "← Back to Gap Analysis" link, 13px DM Sans 500 `--accent`
- Heading: 28px Clash Display 500 "[Skill] — [Topic]" (e.g. "React — Hooks")
- Subheading: "Associates with this gap · sorted by score ascending" — 14px DM Sans `--muted`
- Table: 3 columns — Associate Name (linked to `/trainer/[slug]`), Gap Score, Last Session
- Associate name: 14px DM Sans 400, `--accent` link color on hover with underline
- Gap Score: inline colored indicator matching GapAnalysisTable color logic above
- Last Session: relative timestamp, 14px DM Sans `--muted`

Source: CONTEXT.md D-11

### CalibrationPage (Calibration page)
- Page layout: two sections stacked vertically, max-width 800px
- Heading: 28px Clash Display 500 "Calibration"
- Section 1 — Override Frequency:
  - Card surface: `var(--surface-muted)`, `border: 1px solid var(--border)`, `border-radius: 8px`, `padding: 16px`
  - Large number: 28px Clash Display 600 showing percentage (e.g. "23%")
  - Label: 12px DM Sans 500 `--muted` "of questions overridden by trainer"
  - Subtext: 14px DM Sans 400 `--muted` "X overrides out of Y scored questions"
- Section 2 — Delta Distribution:
  - Section heading: 16px DM Sans 500 "Score Delta Distribution (AI → Trainer)"
  - Chart: `<BarChart>` with 7 bars (buckets: -3, -2, -1, 0, +1, +2, +3)
  - `<ResponsiveContainer width="100%" height={160}>`
  - X-axis: bucket labels (-3 through +3), 12px DM Sans 500 `--muted`
  - Y-axis: count, hidden axis line, 4 ticks, 11px DM Sans `--muted`
  - Grid: horizontal dotted lines, `stroke: var(--border)`, `strokeDasharray="2 4"`
  - Bar fill: `--danger` for negative buckets, `var(--surface-muted)` + `--border` outline for zero, `--success` for positive
  - Tooltip: "X overrides at delta [N]", `background: var(--surface)`, `border: 1px solid var(--border)`
  - Legend note: 12px DM Sans `--muted` below chart: "Positive delta = trainer raised AI score. Negative = trainer lowered."
- Empty state (no sessions with override data): see Copywriting Contract

Source: CONTEXT.md D-16, D-17, D-18, D-19

## Interaction States

| Surface | State | Visual Treatment |
|---------|-------|-----------------|
| KPI card | Loading | `--surface-muted` skeleton block, 150ms fade-in on data arrival |
| KPI card | Loaded | Numbers fade in 150ms ease-out |
| KPI card | Error | Replace number with "—" in `--muted` |
| Roster sparkline | Loading | Empty 56×20 gray block |
| Roster sparkline | Loaded | No animation (`isAnimationActive={false}`) |
| Roster row | Hover | `background: var(--highlight)`, 150ms ease-out |
| Gap Analysis row | Default | `border-bottom: 1px solid var(--border-subtle)` |
| Gap Analysis row | Hover | `background: var(--highlight)`, cursor pointer, 150ms ease-out |
| Gap Analysis sort header | Active | `--accent` arrow icon, `--ink` header text |
| Gap drill-through link | Default | `--ink` |
| Gap drill-through link | Hover | `--accent`, underline |
| Calibration bar | Hover | Tooltip appears (Recharts default), slight opacity lift |
| Cohort Trends line | Hover | Tooltip appears at nearest point |
| Any analytics component | Cohort switch | Re-fetch with new `?cohort=<id>` param, skeleton state while loading |

Focus states (keyboard nav, consistent with P21):
- All interactive elements: `outline: 2px solid var(--accent); outline-offset: 2px` on `:focus-visible`

Source: DESIGN.md §Motion, CONTEXT.md discretion (loading states), 21-UI-SPEC.md §Interaction States
