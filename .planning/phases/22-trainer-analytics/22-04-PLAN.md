1. Create `src/app/api/trainer/calibration/route.ts`:
   - Auth guard: trainer/admin only
   - Parse optional `?cohort=<id>`
   - Fetch sessions with assessments JSON (per D-19 — compute per-question deltas from Session.assessments):
     ```typescript
     const sessions = await prisma.session.findMany({
       where: {
         status: 'completed',
         ...(cohortId ? { associate: { cohortId } } : {}),
       },
       select: {
         assessments: true,
         aiTrainerVariance: true,
       },
     })
     ```
   - For each session, iterate over assessments (parsed from JSON):
     - Count questions where both `llmScore` and `finalScore` exist (totalScoredQuestions)
     - Count questions where `finalScore !== llmScore` (overrideCount)
     - Compute delta = `Math.round(finalScore - llmScore)`, clamp to [-3, +3]
     - Increment the appropriate deltaBucket
   - Compute `overrideRate = totalScoredQuestions > 0 ? (overrideCount / totalScoredQuestions) * 100 : null`
   - Initialize deltaBuckets: `{ "-3": 0, "-2": 0, "-1": 0, "0": 0, "1": 0, "2": 0, "3": 0 }`
   - Return `CalibrationData`

   Note: Using `prisma.session.findMany` + application-layer computation (not $queryRaw) because assessments is a JSON column that needs per-question iteration. This is appropriate because calibration is a low-traffic trainer page.

2. Create test file with vitest mocking identity and prisma.session.findMany.
