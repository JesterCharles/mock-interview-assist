1. Create `src/app/api/trainer/gap-analysis/route.ts` with a single GET handler that serves both aggregation and drill-through based on query params:

**Aggregation mode** (no `skill` param):
   - Auth guard: trainer/admin
   - Parse optional `?cohort=<id>` (parseInt, reject non-numeric)
   - `prisma.$queryRaw` (per D-10, RESEARCH.md Pattern 9):
     ```sql
     SELECT gs.skill, gs.topic,
       COUNT(DISTINCT gs."associateId") as associates_affected,
       AVG(gs."weightedScore") as avg_gap_score
     FROM "GapScore" gs
     JOIN "Associate" a ON a.id = gs."associateId"
     WHERE gs.topic != ''
       [AND a."cohortId" = $1]
     GROUP BY gs.skill, gs.topic
     ORDER BY associates_affected DESC, avg_gap_score ASC
     ```
   - Convert bigint COUNT with `Number()`
   - Return `GapAnalysisRow[]`

**Drill-through mode** (has `skill` and `topic` params):
   - Auth guard: same
   - Parse `skill` and `topic` from search params (string, validated non-empty)
   - Parse optional `?cohort=<id>`
   - `prisma.$queryRaw` (per RESEARCH.md Pattern 9 drill-through):
     ```sql
     SELECT a.slug, a."displayName", gs."weightedScore" as gap_score,
       (SELECT MAX(s."createdAt") FROM "Session" s WHERE s."associateId" = a.id) as last_session
     FROM "GapScore" gs
     JOIN "Associate" a ON a.id = gs."associateId"
     WHERE gs.skill = $1 AND gs.topic = $2
       [AND a."cohortId" = $3]
     ORDER BY gs."weightedScore" ASC
     ```
   - Return `GapDrillThroughRow[]`

2. Create test file mocking identity and prisma.
