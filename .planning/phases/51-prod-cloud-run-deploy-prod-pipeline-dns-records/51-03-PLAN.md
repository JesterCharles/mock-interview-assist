**Complete `deploy-prod.yml`:**

```yaml
name: Deploy to Prod

on:
  push:
    tags:
      - 'v*'   # D-11, D-13: prod deploys triggered ONLY by SemVer-prefixed tags

concurrency:
  group: deploy-prod
  cancel-in-progress: false   # serialize prod deploys; no racing concurrent tags

permissions:
  id-token: write    # WIF requirement — D-12
  contents: read

jobs:
  deploy:
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Authenticate to GCP (WIF)
        uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: projects/${{ vars.PROD_PROJECT_NUMBER }}/locations/global/workloadIdentityPools/github-actions/providers/github
          service_account: github-actions-deployer@nlm-prod.iam.gserviceaccount.com

      - name: Setup gcloud
        uses: google-github-actions/setup-gcloud@v2
        with:
          project_id: nlm-prod

      - name: Fetch secrets from Secret Manager
        id: secrets
        uses: google-github-actions/get-secretmanager-secrets@v2
        with:
          secrets: |-
            DIRECT_URL:nlm-prod/DIRECT_URL
            ADMIN_EMAILS:nlm-prod/ADMIN_EMAILS

      - name: Setup Docker buildx
        uses: docker/setup-buildx-action@v3

      - name: Configure Docker for Artifact Registry
        run: gcloud auth configure-docker us-central1-docker.pkg.dev --quiet

      - name: Build and push image
        id: build
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: |
            us-central1-docker.pkg.dev/nlm-prod/nlm-app/nlm-app:${{ github.ref_name }}
            us-central1-docker.pkg.dev/nlm-prod/nlm-app/nlm-app:latest
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Extract image digest
        id: digest
        run: |
          DIGEST="${{ steps.build.outputs.digest }}"
          echo "Built digest: $DIGEST"
          echo "digest=$DIGEST" >> "$GITHUB_OUTPUT"

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci --prefer-offline

      - name: Generate Prisma client
        run: npx prisma generate

      - name: Prisma migrate deploy (prod DB)
        env:
          DIRECT_URL: ${{ steps.secrets.outputs.DIRECT_URL }}
        run: npx prisma migrate deploy

      - name: Deploy to Cloud Run (by digest, per INFRA-02)
        uses: google-github-actions/deploy-cloudrun@v2
        with:
          service: nlm-prod
          image: us-central1-docker.pkg.dev/nlm-prod/nlm-app/nlm-app@${{ steps.digest.outputs.digest }}
          region: us-central1
          project_id: nlm-prod

      - name: Post-deploy smoke (pre-DNS; *.run.app URL)
        run: |
          URL=$(gcloud run services describe nlm-prod --region=us-central1 --project=nlm-prod --format='value(status.url)')
          echo "smoke target: $URL/api/health"
          HTTP=$(curl -sf -o /dev/null -w '%{http_code}' --max-time 30 "$URL/api/health" || echo "000")
          echo "HTTP: $HTTP"
          case "$HTTP" in
            200|503) echo "smoke OK ($HTTP — 503 accepted per INFRA-07 Prisma transient tolerance)" ;;
            *) echo "smoke FAIL: $HTTP"; exit 1 ;;
          esac

      - name: Post-deploy revision tag
        run: |
          REVISION=$(gcloud run services describe nlm-prod --region=us-central1 --project=nlm-prod --format='value(status.latestReadyRevisionName)')
          echo "Deployed revision: $REVISION"
          echo "Tag: ${{ github.ref_name }}"
          echo "Digest: ${{ steps.digest.outputs.digest }}"

      - name: Email on failure
        if: failure()
        uses: dawidd6/action-send-mail@v3
        with:
          server_address: smtp.resend.com
          server_port: 465
          secure: true
          username: resend
          password: ${{ secrets.RESEND_API_KEY }}
          subject: "[NLM] deploy-prod.yml FAILED (${{ github.ref_name }})"
          to: ${{ steps.secrets.outputs.ADMIN_EMAILS }}
          from: alerts@nextlevelmock.com
          body: |
            Prod deploy failed. Tag: ${{ github.ref_name }}
            Run: ${{ github.server_url }}/${{ github.repository }}/actions/runs/${{ github.run_id }}
```

**Key guardrails to preserve:**
- `concurrency.group: deploy-prod` + `cancel-in-progress: false` — two tags pushed in quick succession serialize, never race (avoids half-deployed state).
- `timeout-minutes: 15` — prod deploy should never exceed CI-06's 5-min target × 3 safety margin.
- No `GOOGLE_APPLICATION_CREDENTIALS` env var (WIF only per D-12 + CI-04).
- No `:latest` on Cloud Run deploy (only build-push uses `:latest` as a convenience tag; Cloud Run still pulls by `@digest`).

**Trigger the first run:**
1. Commit `deploy-prod.yml` to the branch
2. Merge to `main` (normal PR process — `pr-checks.yml` runs)
3. After merge, tag + push: `git tag -a v1.5.0-rc1 -m "Phase 51 prod pipeline smoke"` + `git push origin v1.5.0-rc1`
4. Wait for run to finish: `gh run watch $(gh run list --workflow=deploy-prod.yml --limit 1 --json databaseId -q '.[0].databaseId')`
5. Assert: `gh run list --workflow=deploy-prod.yml --limit 1 --json conclusion -q '.[].conclusion'` equals `success`.

**Rollback if first run fails:**
- Delete remote tag: `git push --delete origin v1.5.0-rc1`
- Delete local tag: `git tag -d v1.5.0-rc1`
- Fix workflow + retry with fresh tag (`v1.5.0-rc2`)
- Do NOT edit and re-push the same tag — tags are immutable in the deploy contract.

Do NOT add:
- PR/branch triggers (violates D-13 — prod is tag-only)
- Canary traffic splits (deferred in CONTEXT)
- Auto-rollback on health-check fail (deferred in CONTEXT)
- Manual approval gate (solo-dev scope — tag push IS the approval)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → GCP | WIF OIDC exchange (short-lived token; no long-lived JSON key) |
| CI runner → prod Supabase | `prisma migrate deploy` connects via DIRECT_URL pulled from Secret Manager (bypasses pooler) |
| CI runner → prod Artifact Registry | Pushes signed image tagged with git ref + digest |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-51-03 | Tampering | Prod deploy pipeline triggered by untested code (direct main push) | mitigate | Workflow triggers ONLY on `push.tags: ['v*']`; no branch/PR trigger; tag push by trainer is the approval gate; `pr-checks.yml` already gates all merges to main before a tag can exist on a main commit |
| T-51-12 | Elevation of Privilege | Compromised GH Actions runner escalates to GCP owner | mitigate | WIF provider D-16 attribute condition restricts federation to `attribute.repository == "JesterCharles/mock-interview-assist"`; D-18 role matrix grants only `artifactregistry.writer`, `run.admin`, `iam.serviceAccountUser` — NOT `roles/owner` or `roles/editor`; impersonated SA is `github-actions-deployer@nlm-prod`, not the runtime `nlm-cloudrun-sa` |
| T-51-13 | Tampering | Attacker re-pushes an existing tag pointing at a malicious commit | accept | Git tags are immutable on GitHub server-side once pushed; reusing a tag requires force-push which triggers visible audit events; solo-dev account has 2FA; further mitigation deferred (audit log review cadence) |
| T-51-14 | Information Disclosure | `prisma migrate deploy` logs contain prod DATABASE_URL/DIRECT_URL | mitigate | Secret pulled via `google-github-actions/get-secretmanager-secrets@v2` which auto-masks the value in GH Actions logs; Prisma CLI redacts connection strings in output; workflow env-var scoping limits leak surface |
| T-51-15 | Denial of Service | Two simultaneous tag pushes deploy races | mitigate | `concurrency.group: deploy-prod` + `cancel-in-progress: false` serializes prod deploys; second run waits for first to complete |
| T-51-16 | Repudiation | Who tagged v1.5.0-rc1 is unclear | accept | `github.actor` captured in workflow logs; tag messages include trainer name/email via `git config`; GitHub tag signing out of v1.5 scope (manual flow — solo-dev) |
