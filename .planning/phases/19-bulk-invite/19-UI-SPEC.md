## Interaction Contract

### Screen 1 — Input Form (`/trainer/onboarding`)

**Textarea**
- Variant: multi-line, `--surface` background, `--border` 1px solid border, 8px border-radius, 16px padding
- Min height: 120px, max height: 240px (scrollable)
- Resize: vertical only
- Font: 13px DM Sans 400, `--ink` color
- Placeholder: `--muted` color
- Focus: `--accent` 2px outline, offset 2px
- Live parsing fires on `onChange` (debounced 150ms)

**Chip list**
- Renders below textarea as a wrapping flex row (gap: 8px, row-gap: 8px)
- Each chip: border-radius 9999px (pill), 6px vertical + 12px horizontal padding, 13px DM Sans 400
- Chip colors: per semantic table above (valid/invalid/duplicate)
- X button: minimum 44×44px hit area, `×` glyph, `aria-label="Remove {email}"` (e.g. `aria-label="Remove user@example.com"`), same text color as chip, positioned inline-end
- Chip list has max-height 160px with overflow-y: auto when count exceeds ~20 chips

**Count summary**
- Renders immediately below chip list
- Format: "{N} valid · {N} invalid · {N} duplicate"
- 12px DM Sans 600
- valid count: `--success` color, invalid: `--danger`, duplicate: `--warning`

**Cap error**
- Inline below count summary when valid count > 50
- 13px DM Sans 400, `--danger` color
- "Preview Invites" button disabled when cap exceeded

**Cohort dropdown**
- Single `<select>` element styled to match `--surface` + `--border`
- min-height 44px, 8px border-radius, 16px padding
- Required — "Preview Invites" disabled when no cohort selected
- Options: each cohort name, sorted alphabetically
- Placeholder option "Select a cohort" is disabled/value=""
- "No cohorts" state: dropdown replaced by inline error copy (see copywriting table)

**"Preview Invites" button**
- `.btn-accent-flat` class from globals.css
- Disabled when: textarea empty OR no cohort selected OR valid count = 0 OR valid count > 50
- Loading state: spinner (16px, `--surface` color) replaces label text during roster fetch

### Screen 2 — Preview Table

**Transition**
- Input form fades out (fade-in 150ms), preview section fades in (slide-up 200ms)
- URL stays at `/trainer/onboarding` (no route change — SPA state transition)

**Preview summary line**
- 16px DM Sans 400, `--muted` color
- "{N} will be invited · {N} will be reassigned · {N} will be skipped"

**Preview table**
- Full-width, `--surface` background, `--border` 1px table borders
- Row hover: `--highlight` background (150ms ease-out transition)
- Columns: Checkbox | Email | Action | Notes
- Checkbox column: 40px wide; only actionable rows (New, Reassign) are checkable; Skip rows: checkbox disabled + grayed
- Email column: 13px DM Sans 400, `--ink`
- Action column: semantic badge (pill shape, 12px DM Sans 600, per badge table above)
- Notes column: 12px DM Sans 400, `--muted` (e.g. "Currently in Cohort B", "Invited 2 min ago")
- Default: all New + Reassign rows checked; Skip rows unchecked + disabled

**"Confirm & Send {N} Invites" button**
- `.btn-accent-flat` from globals.css
- Label updates live as trainer checks/unchecks rows: "Confirm & Send 12 Invites"
- Disabled when actionable checked count = 0
- Loading state: spinner replaces label; all checkboxes + "Back to Edit" disabled during send

**"Back to Edit" button**
- `.btn-secondary-flat` from globals.css
- Returns to Screen 1 with textarea content and cohort selection preserved

### Screen 3 — Results Table

**Transition**
- Preview table replaced by results table in-place (fade-in 150ms)
- Toast fires via `react-hot-toast` immediately after table renders

**Results summary**
- "{N} invited · {N} skipped · {N} failed" — same format as preview, counts from actual results

**Results table**
- Same layout as preview table: Email | Status | Notes (no checkbox column)
- Status badge colors per result badge table above
- Notes: error message for failed rows (12px, `--danger` color)
- Sorted: invited first, then reassigned, then skipped, then failed

**Toast**
- Position: bottom-right (react-hot-toast default)
- Duration: 4000ms
- All success: success toast (green bar, white surface) — "All invites sent."
- Partial failure: warning toast (orange bar) — "{N} failed. Check the results table."
- Style: DM Sans 13px 400, `--ink` text, `--surface` background, `--border` border

**"Invite More" button**
- `.btn-secondary-flat` from globals.css
- Resets form to Screen 1 (clears textarea, preserves cohort selection)

### Loading States

| State | Treatment |
|-------|-----------|
| Roster fetch (preview click) | Button spinner (16px) + "Loading…" label; textarea + dropdown remain interactive |
| Bulk send (confirm click) | Button spinner + "Sending…"; table rows grayed at 50% opacity; "Back to Edit" disabled |

### Error States

| Trigger | Treatment |
|---------|-----------|
| Cap exceeded (>50 valid) | Inline error below count summary; button disabled |
| Daily limit 429 from API | Banner above table: danger background, "Would exceed daily limit…" copy; results table NOT shown |
| Network error on roster fetch | Inline error below button: "Failed to load roster. Retry." with retry button |
| All emails failed | Results table shown; toast: warning; no invites were sent |
