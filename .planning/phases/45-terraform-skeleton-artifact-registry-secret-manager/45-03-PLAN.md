```hcl
# Source: 45-RESEARCH.md §Pattern 4; D-09 (13 secrets); D-10 (TF owns shells only — no values).

# Values are populated out-of-band via `gcloud secrets versions add` (see iac/cloudrun/README.md).

  }

  labels = {

}
```

```hcl
# Phase 45 — iac/cloudrun/iam.tf
# Source: 45-RESEARCH.md §Pattern 5; D-11 (2 SAs per project); D-13 (account IDs); D-14 (accessor now, WIF deferred to P48).
# T-45-04 mitigation: per-secret binding via for_each — no project-level secretmanager.admin grant.
# T-45-05 mitigation: no google_service_account_key resources — only SYSTEM_MANAGED keys exist on SAs.

resource "google_service_account" "cloudrun" {
  project      = var.project_id
  account_id   = var.cloudrun_sa_id
  display_name = "NLM Cloud Run runtime identity (${var.env})"
  description  = "Attached to the NLM Cloud Run service in Phase 47. Reads secrets listed in var.secret_names."

  depends_on = [google_project_service.apis]
}

resource "google_service_account" "ghactions" {
  project      = var.project_id
  account_id   = var.ghactions_sa_id
  display_name = "NLM GitHub Actions deployer (${var.env})"
  description  = "WIF principal bound in Phase 48. No role bindings in Phase 45."

  depends_on = [google_project_service.apis]
}

# Per-secret accessor binding — strictly scoped, no project-wide grant.
# for_each over the secret set keeps binding count == secret count (13 per project).
resource "google_secret_manager_secret_iam_member" "cloudrun_accessor" {
  for_each = google_secret_manager_secret.app

  project   = var.project_id
  secret_id = each.value.secret_id
  role      = "roles/secretmanager.secretAccessor"
  member    = "serviceAccount:${google_service_account.cloudrun.email}"
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Terraform → Secret Manager | TF creates secret shells only; plaintext never crosses this boundary (T-45-01 mitigation) |
| `nlm-cloudrun-sa` → Secret Manager | Access scoped per-secret via `secretAccessor`; no admin, no project-level grant (T-45-04) |
| `github-actions-deployer` → GCP | No bindings in Phase 45; WIF (OIDC) binding deferred to Phase 48 (T-45-05) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-01 | Information Disclosure | tfstate file (Secret Manager values) | mitigate | HCL contains only `google_secret_manager_secret` (shell) — NO `google_secret_manager_secret_version`. Values are added out-of-band via `gcloud secrets versions add` per D-10. Verified by grep assertion: file must NOT match `google_secret_manager_secret_version`. |
| T-45-04 | Elevation of Privilege | `nlm-cloudrun-sa` | mitigate | Per-secret `google_secret_manager_secret_iam_member` via `for_each = google_secret_manager_secret.app`. No `google_project_iam_member` for `secretmanager.*`. Grant limited to the exact 13 secrets. Future secrets (Phase 50 Judge0) must be added to the for_each set explicitly. |
| T-45-05 | Spoofing | SA credentials | mitigate | Zero `google_service_account_key` resources in HCL. `github-actions-deployer` has no bindings. WIF (OIDC) in Phase 48 replaces keys. gcloud assertion: `keys list` shows only SYSTEM_MANAGED. |
</threat_model>
