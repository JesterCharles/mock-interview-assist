  <name>Task 1: Configure next.config.ts, Dockerfile, and docker-compose.yml for Prisma</name>
  <files>next.config.ts, Dockerfile, docker-compose.yml</files>
  <read_first>

    - docker-compose.yml (current compose — update healthcheck)
    - .planning/phases/01-db-foundation/01-RESEARCH.md (Pattern 3: outputFileTracingIncludes, Pattern 4: Dockerfile changes)
  </read_first>
  <action>
1. Update `next.config.ts` to include Prisma binaries in standalone output (PERSIST-06):
   ```typescript
   import type { NextConfig } from "next";

   const nextConfig: NextConfig = {
     output: 'standalone',
     outputFileTracingIncludes: {
       '/': [
         './node_modules/prisma/**/*',
         './node_modules/@prisma/**/*',
         './node_modules/.bin/**/*',
       ],
     },
   };

   export default nextConfig;
   ```
   Three paths are required per RESEARCH.md Pattern 3:
   - `prisma/**/*` — CLI package
   - `@prisma/**/*` — Engine binaries and adapter packages
   - `.bin/**/*` — Executable symlinks for npx prisma discovery

2. Update `Dockerfile` Builder stage — add `prisma generate` BEFORE `npm run build`:
   After the `COPY . .` line in the Builder stage, add:
   ```dockerfile
   # Generate Prisma client before building (schema must be present)
   RUN npx prisma generate
   ```
   The full Builder stage should be:
   ```dockerfile
   FROM node:22-alpine AS builder
   WORKDIR /app
   COPY --from=deps /app/node_modules ./node_modules
   COPY . .
   RUN npx prisma generate
   RUN npm run build
   ```

3. Update `Dockerfile` HEALTHCHECK to use /api/health (per D-02):
   Replace the existing HEALTHCHECK line:
   ```dockerfile
   HEALTHCHECK --interval=30s --timeout=10s --start-period=15s --retries=3 \
       CMD wget --no-verbose --tries=1 --spider http://localhost:3000/api/health || exit 1
   ```
   Note: `start-period` increased from 5s to 15s — Prisma's first connection to Supabase can take a few seconds on cold start.

4. Update `docker-compose.yml` healthcheck to match:
   ```yaml
   healthcheck:
     test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:3000/api/health"]
     interval: 30s
     timeout: 10s
     retries: 3
     start_period: 15s
   ```

5. Do NOT add `prisma migrate deploy` to the Dockerfile CMD or entrypoint. Per D-03, migrations run as a separate command outside the container. The container only runs the Next.js server.
  </action>
