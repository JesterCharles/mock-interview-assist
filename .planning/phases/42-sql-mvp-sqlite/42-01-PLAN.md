**Public API:**

```typescript
import type { SqlTestCase } from './coding-bank-schemas';

export interface NormalizationFlags {
  trimMode: 'strict' | 'normalize';        // default 'normalize'
  numericCoerce: boolean;                    // default true
  orderSensitiveColumns: boolean;            // default true (columns ARE ordered unless trainer relaxes)
  rowOrderSensitive: boolean;                // default false (rows NOT ordered unless trainer opts in)
}

export interface NormalizationResult {
  passed: boolean;
  actualRows: Array<Array<string | number | null>>;
  reason?: string;
}

export function normalizeSqliteResult(
  stdout: string,
  testCase: Pick<SqlTestCase, 'expectedRows' | 'expectedColumns' | 'expectedStdout' | 'trimMode' | 'numericCoerce' | 'orderSensitiveColumns' | 'rowOrderSensitive'>,
): NormalizationResult;
```

**Implementation — 5 steps per D-05, in order:**

1. **Parse stdout** — split on `\n`, filter empty trailing lines, split each line on `\t`. This requires the submit route to have prefixed `.mode tabs\n.headers off\n` to the query (Task 3 guarantees this). If `expectedRows` is absent, short-circuit to `expectedStdout` string comparison fallback (Test 7).

2. **Trim whitespace per cell** — per `trimMode`:
   - `'strict'` — trim leading + trailing only (preserves internal whitespace)
   - `'normalize'` (default) — trim + collapse internal runs of whitespace to single space
   - Apply to both expected and actual cells symmetrically.

3. **Numeric coercion** — if `numericCoerce` is true (default), try `Number(cell)` on every cell on BOTH sides; if both parse to finite numbers, compare as numbers. Empty-string cells map to `null` (Test 8). String cells that don't parse stay as strings.

4. **Column reorder** — if `expectedColumns` supplied AND `orderSensitiveColumns === false`, sort both the expected and actual column arrays alphabetically, then reorder each row's cells by the new column order before compare. If actual column count mismatches expected column count at this stage, short-circuit to `{ passed: false, reason: \`column count mismatch: expected \${N}, got \${M}\` }`.

   Note: SQLite `.mode tabs` + `.headers off` outputs data WITHOUT a header row. When `expectedColumns` is provided, the normalizer ASSUMES actual columns appear in the same declarative order the trainer used when writing the query. The `orderSensitiveColumns` flag only affects whether we also accept alphabetical-reorder matching.

5. **Row sort** — if `rowOrderSensitive === false` (default for SQL), sort both expected and actual rows using a deterministic lexicographic comparator over cells-as-strings, then compare row-by-row. If `rowOrderSensitive === true`, compare in original order.

Return `{ passed: true, actualRows }` on success OR `{ passed: false, actualRows, reason: '...' }` on first failure.

**Defaults** — always apply these when flag is `undefined`:
```typescript
const resolvedFlags = {
  trimMode: testCase.trimMode ?? 'normalize',
  numericCoerce: testCase.numericCoerce ?? true,
  orderSensitiveColumns: testCase.orderSensitiveColumns ?? true,
  rowOrderSensitive: testCase.rowOrderSensitive ?? false,
};
```

```typescript
    // For non-SQL: tc.stdin is piped to Judge0 as stdin. For SQL: tc.stdin is
    // concatenated into the source; we pass empty stdin to Judge0.
    //
    // We DO NOT pass expected_output to Judge0 for SQL — we normalize server-side
    // per D-06 using sqlResultNormalizer.
  }),
);
```
