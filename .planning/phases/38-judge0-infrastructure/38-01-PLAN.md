<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host network ↔ Judge0 container | Submissions originate from app; sandboxed code runs inside judge0-workers. Must never escape container or reach host. |
| Public internet ↔ judge0-server:2358 | Judge0 HTTP API must NEVER be reachable from the public internet. |
| Sandboxed code ↔ network | Code submitted by associates must not make outbound network calls. |
| docker-compose env_file ↔ git | Secrets must never be committed; only `.example` templates. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-01 | Elevation of Privilege | judge0-workers sandbox (isar) | mitigate | Pin image to 1.13.1 — patches GHSA-q7vg-26pg-v5hr sandbox escape (D-01). Digest comment in compose for reproducibility. |
| T-38-02 | Information Disclosure | judge0-server HTTP API | mitigate | Bind 2358 to 127.0.0.1 only, never 0.0.0.0 (D-09). Place services on internal `judge0-net` bridge; only app container bridges to it (D-08). Comment in compose warns against 0.0.0.0 binding. |
| T-38-03 | Information Disclosure / Tampering | Sandboxed code → external hosts | mitigate | `ENABLE_NETWORK=false` set on both server and workers (D-04). Locked at image config layer — sandbox honors regardless of submission content. |
| T-38-04 | Denial of Service | Fork bomb / infinite loop / memory hog | mitigate | Per-process CPU (`MAX_CPU_TIME_LIMIT=10`), memory (`MAX_MEMORY_LIMIT=256000`), and process (`MAX_PROCESSES_AND_OR_THREADS=60`) caps (D-05, D-06). Per-container resource.limits cap cumulative impact (D-03 placeholder; 38-03 commits final). `COUNT_WORKERS` knob caps parallelism (D-07). |
| T-38-05 | Spoofing / Tampering | Judge0 API without auth | mitigate | `AUTHN_TOKEN` required on every request via `X-Auth-Token` header (D-10). Shared via env_file, never hardcoded. |
| T-38-06 | Tampering | Unpinned image drifts to vulnerable version | mitigate | Exact tag pin for judge0 (1.13.1), postgres (15-alpine), redis (7-alpine) per D-01, D-02. Digest comments document target sha256. |
| T-38-07 | Information Disclosure | Secrets committed to repo | mitigate | `.env.judge0` in .gitignore; only `.env.judge0.example` with placeholder values is committed. Verification runs `git check-ignore .env.judge0`. |
</threat_model>
