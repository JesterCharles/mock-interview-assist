<!-- /api/coding/challenges contract (Phase 39 D-13/D-14): -->

Query params (all optional):
- `cursor=<cuid>` — pagination cursor
- `limit=<number>` — default 20, max 100
- `language=python|javascript|typescript|java|sql|csharp`
- `difficulty=easy|medium|hard`
- `status=unstarted|attempted|passed`
- `week=<number>`

Response shape:
```typescript
interface ChallengeListItem {
  id: string;                 // cuid
  slug: string;
  title: string;
  language: string;           // one of the 6
  difficulty: 'easy' | 'medium' | 'hard';
  skillSlug: string;          // matches curriculum
  cohortId: string | null;
  latestAttempt?: {
    submittedAt: string;      // ISO
    verdict: 'pass' | 'fail' | 'timeout' | 'mle' | 'runtime_error' | 'compile_error' | 'pending';
  } | null;
}

interface ChallengeListResponse {
  items: ChallengeListItem[];
  nextCursor: string | null;
}
```

Derive status badge from `latestAttempt`:
- `null` → unstarted
- `verdict === 'pass'` → passed
- else → attempted

<!-- Error envelope: -->
```typescript
{ error: { code: 'AUTH_REQUIRED' | 'FORBIDDEN' | 'RATE_LIMITED' | ..., message: string } }
```
