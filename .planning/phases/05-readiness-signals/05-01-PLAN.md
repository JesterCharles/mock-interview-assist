Tests should mock prisma.gapScore.findMany, prisma.session.count, prisma.session.findMany, prisma.associate.findMany, and prisma.associate.update.

    Then create src/lib/readinessService.ts implementing:

    1. **Type definitions** at top of file:
       ```typescript
       export type ReadinessStatus = 'ready' | 'improving' | 'not_ready';
       export interface ReadinessResult {
         status: ReadinessStatus;
         recommendedArea: string | null;
         lastComputedAt: Date;
       }
       ```

    2. **computeTrend(associateId: string)** — per D-01:
       - Query last 3 sessions ordered by createdAt DESC, select overallScore
       - If < 3 sessions, return -1
       - Reverse to chronological order, compute simple linear regression slope over [0, 1, 2]
       - Slope formula: numerator = sum((x - xMean) * (y - yMean)), denominator = sum((x - xMean)^2)
       - Return slope (positive = improving, zero = flat, negative = declining)

    3. **computeReadiness(associateId: string, threshold: number)** — per D-01, D-02, D-03:
       - Count sessions for associate. If < 3: return { status: 'not_ready', recommendedArea: null, lastComputedAt: now }
       - Fetch skill-level GapScores (where topic IS null) for the associate
       - Compute weighted average across all skill-level scores
       - Call computeTrend(associateId) for trend slope
       - Classification cascade (ORDER MATTERS per Pitfall 1):
         a. avg >= threshold AND trend >= 0 -> 'ready'
         b. trend > 0 (strictly positive, per Open Question 3) AND avg < threshold -> 'improving'
         c. Everything else -> 'not_ready'
       - Recommended area (per D-03): query GapScore where topic IS NOT null, order by weightedScore ASC, take first. Fall back to lowest skill if no topics. Null if < 3 sessions.
       - Return ReadinessResult

    4. **recomputeAllReadiness(threshold: number)** — per D-04:
       - Fetch all associate IDs
       - For each: call computeReadiness, then update Associate record with result fields
       - Add code comment: "Synchronous batch — safe for MVP (<200 associates). If scale grows, move to background task."

    5. **updateAssociateReadiness(associateId: string, threshold: number)** — convenience wrapper:
       - Call computeReadiness, then prisma.associate.update with readinessStatus, recommendedArea, lastComputedAt
       - This is what the session save pipeline calls

    IMPORTANT per Pitfall 5: If the Session model does not have a scalar overallScore field, compute it from the assessments JSON. Check the schema first. If assessments is a JSON column, parse it and average the scores to derive overallScore in application code within computeTrend.

    Use the Prisma singleton from src/lib/prisma.ts (created in Phase 1).
