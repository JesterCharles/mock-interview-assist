<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> PDF routes | Unauthenticated users must not access PDF data (names, scores, readiness) |
| URL params -> Prisma | cohort ID and slug params cross trust boundary into DB queries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-02 | Information Disclosure | PDF routes expose PII (names, scores) | mitigate | `getCallerIdentity()` trainer/admin guard at route entry — same pattern as all `/api/trainer/*` routes |
| T-24-03 | Tampering | `?cohort=<id>` injection | mitigate | `Number.parseInt()` + `Number.isInteger()` guard; Prisma parameterized `$queryRaw` for all queries |
| T-24-04 | Tampering | `?slug=<slug>` injection | mitigate | Prisma parameterized query (`Prisma.sql` template literal); no raw string interpolation |
| T-24-05 | Denial of Service | Large cohort PDF generation | accept | Low risk in single-trainer deploy; cohort size bounded by manual onboarding (50/batch). Acceptable for MVP. |
</threat_model>
