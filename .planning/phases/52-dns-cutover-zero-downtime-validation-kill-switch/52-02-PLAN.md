  <what-built>
    Claude runs a 10-iteration polling loop (every 30s, 5 minutes total) observing DNS propagation on 1.1.1.1 and `/api/health` response from the apex. Every iteration appends a timestamped block to the cutover log.

    Exact command (Claude drives, operator watches for any 502/503):

    ```bash
    LOG=.planning/cutover-log-v1.5.md
    echo "" >> "$LOG"
    echo "## Cutover Execution (T-0 to T+5min)" >> "$LOG"

    for i in $(seq 0 9); do
      TS=$(date -u +%Y-%m-%dT%H:%M:%SZ)
      echo "" >> "$LOG"
      echo "### Poll $i (T+${i}x30s) — $TS" >> "$LOG"

      echo "[$TS] \$ dig @1.1.1.1 +short nextlevelmock.com A" >> "$LOG"
      dig @1.1.1.1 +short nextlevelmock.com A >> "$LOG"

      echo "[$TS] \$ curl -sfI https://nextlevelmock.com/api/health" >> "$LOG"
      # -w captures the effective status code even when -f would exit nonzero.
      curl -sI -o /tmp/curl_body.$$ -w 'HTTP %{http_code} via IP=%{remote_ip}\n' \
        https://nextlevelmock.com/api/health >> "$LOG" 2>&1

      # Extract status code for hard-fail detection.
      CODE=$(curl -s -o /dev/null -w '%{http_code}' https://nextlevelmock.com/api/health)
      echo "[$TS] status=$CODE" >> "$LOG"

      # Hard fail trigger per phase constraints: 502/503 during T-0..T+5min = rollback required.
      if [ "$CODE" = "502" ] || [ "$CODE" = "503" ]; then
        echo "[$TS] !!! HARD FAIL — $CODE observed. TRIGGERING ROLLBACK via kill switch. !!!" >> "$LOG"
        # Operator now runs Plan 03 Task 1 kill-switch revert IMMEDIATELY.
        # Plan 02 stops here; phase cannot complete.
        exit 1
      fi

      [ $i -lt 9 ] && sleep 30
    done

    # T+5min verification (D-14 / §4 from DEPLOY.md runbook).
    TS=$(date -u +%Y-%m-%dT%H:%M:%SZ)
    echo "" >> "$LOG"
    echo "### T+5min Verification — $TS" >> "$LOG"
    echo "[$TS] \$ dig @1.1.1.1 +short nextlevelmock.com A" >> "$LOG"
    FINAL_IP=$(dig @1.1.1.1 +short nextlevelmock.com A | head -1)
    echo "$FINAL_IP" >> "$LOG"
    echo "[$TS] \$ curl -sfI https://nextlevelmock.com/api/health" >> "$LOG"
    curl -sfI https://nextlevelmock.com/api/health | head -5 >> "$LOG"
    echo "[$TS] \$ dig +short legacy.nextlevelmock.com A" >> "$LOG"
    dig @1.1.1.1 +short legacy.nextlevelmock.com A >> "$LOG"
    echo "[$TS] \$ dig +short www.nextlevelmock.com A" >> "$LOG"
    dig @1.1.1.1 +short www.nextlevelmock.com A >> "$LOG"
    ```

    If propagation at 1.1.1.1 takes longer than 5 minutes (T-52-01), Claude continues polling up to 10 minutes with `sleep 60` intervals. Past 10 minutes without resolution = escalation to operator; operator decides rollback or continue.

    Note on Cloudflare proxy: because `proxied=true`, `dig nextlevelmock.com` resolves to a Cloudflare edge IP (not PROD_LB_IP directly). The KEY signal is `curl -sfI .../api/health` returning 200 with a `Server` header consistent with Cloud Run (`Server: Google Frontend`), not the nginx/Docker server v0.1 was running. Record the response headers in the log.
