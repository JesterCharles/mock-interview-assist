<objective>
Split automated-interview completion into two endpoints per Codex finding #3, and add DB-backed readiness recompute markers per Codex finding #5:

- `/api/public/interview/complete` (existing, hardened): strip client-supplied `associateSlug` unconditionally before persistence. Anonymous callers cannot forge associate linkage.
- `/api/associate/interview/complete` (NEW): requires `associate_session` cookie (version-checked via Phase 9). Identity derives ONLY from `getAssociateSession()`. Triggers gap/readiness fan-out.
- `runReadinessPipeline(associateId, sessionId?)` helper: manages the `Session.readinessRecomputeStatus` column (pending → done/failed) so failures are repairable by the sweep (Plan 10-03).

Purpose: Close the loop between authenticated automated interviews and the readiness engine WITHOUT leaving an identity-spoofing hole on the anonymous endpoint, AND without a silent-failure mode for readiness recompute.

Output: Hardened anonymous endpoint, new authenticated endpoint, shared pipeline helper that updates the DB repair marker.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→/api/public/interview/complete | Untrusted body; associateSlug MUST be stripped before persist |
| client→/api/associate/interview/complete | Body + cookie; identity taken ONLY from cookie |
| cookie→server | associate_session (Phase 9) — HMAC + version-checked |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Spoofing | public/complete identity injection | mitigate | Unconditionally null associateSlug before persist (Codex #3). Test 2+3 of Task 2 enforces. |
| T-10-02 | Spoofing | associate/complete identity | mitigate | Identity derived EXCLUSIVELY from getAssociateSession (version-checked). Client associateSlug overridden. Test 4 of Task 3 enforces. |
| T-10-09 | Elevation of Privilege | Stale cookie reaching authenticated endpoint | mitigate | getAssociateSession returns null for stale ver (Phase 9 Plan 09-02); 401 returned |
| T-10-10 | Repudiation | Silent failure of readiness recompute | mitigate | DB-backed readinessRecomputeStatus marker (Codex #5); sweep endpoint (Plan 10-03) repairs |
| T-10-03 | Information Disclosure | Pipeline errors | mitigate | Errors logged server-side only; status column captures failure for sweep |
| T-10-04 | Denial of Service | Unbounded gap/readiness work | mitigate | Fire-and-forget + rate limit caps pipeline invocations |
| T-10-05 | Repudiation | Who completed the session | accept | Cookie-derived associateId is sufficient audit trail for MVP |
| T-10-06 | Elevation of Privilege | Associate→trainer | accept | Associate cookie cannot reach trainer routes (Phase 9 AUTH-03) |
