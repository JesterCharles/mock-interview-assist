Plan 03 runs in the same wave but uses a defense-in-depth approach: the server components in `/coding/*` call `isCodingEnabled()` **directly on the server** (Plan 01's export) to decide ComingSoon vs workspace, avoiding a round-trip fetch to `/api/coding/status` at server render. Only the *client hook* `useCodingStatus` fetches the status endpoint (for client-side interactive components like SubmitBar). If Plan 02 has not finished by Plan 03's test run, the hook's fetch will 404 — which the hook treats as "assume enabled" (fail-open: client already rendered ComingSoon if server said so).

Wait — that creates a race. **Better strategy:** Plan 03 does not fetch the status endpoint at all from the hook during Wave 2. The hook is added but its implementation stubs to call `isCodingEnabled` through a Next.js Route Handler that Plan 03 also owns (the status endpoint is Plan 02's file). **Cleanest solution:** Plan 03 imports `isCodingEnabled` directly in server components (both `/coding` pages), and the client-side `useCodingStatus` hook is scaffolded with a fetch to `/api/coding/status` that Plan 02 will have landed by the time Wave 3 integration tests run. Tests in Plan 03 mock the fetch.

    ```typescript
    /**
     * useCodingStatus — Phase 50 (JUDGE-INTEG-02 / D-06)
     *
     * Client hook that fetches GET /api/coding/status and caches the result
     * for 60 seconds (matching Cache-Control: public, s-maxage=60 from the
     * endpoint).
     *
     * Fail-open: if the fetch errors, returns {status: 'error', enabled: null}.
     * Callers should treat null as "assume enabled" — we'd rather render the
     * workspace and let a subsequent submit return 503 than block users when
     * a transient network hiccup occurs.
     */
    'use client';

    import { useEffect, useState } from 'react';

    export type CodingStatus =
      | { status: 'loading'; enabled: null }
      | { status: 'ready'; enabled: boolean }
      | { status: 'error'; enabled: null };

    const CACHE_TTL_MS = 60_000;
    let moduleCache: { value: boolean; fetchedAt: number } | null = null;

    export function resetCodingStatusCache() {
      moduleCache = null;
    }

    export function useCodingStatus(): CodingStatus {
      const initial: CodingStatus =
        moduleCache && Date.now() - moduleCache.fetchedAt < CACHE_TTL_MS
          ? { status: 'ready', enabled: moduleCache.value }
          : { status: 'loading', enabled: null };

      const [state, setState] = useState<CodingStatus>(initial);

      useEffect(() => {
        if (moduleCache && Date.now() - moduleCache.fetchedAt < CACHE_TTL_MS) {
          setState({ status: 'ready', enabled: moduleCache.value });
          return;
        }
        let aborted = false;
        (async () => {
          try {
            const res = await fetch('/api/coding/status', { credentials: 'include' });
            if (aborted) return;
            if (!res.ok) {
              setState({ status: 'error', enabled: null });
              return;
            }
            const body = (await res.json()) as { enabled?: unknown };
            const enabled = body.enabled === true;
            moduleCache = { value: enabled, fetchedAt: Date.now() };
            setState({ status: 'ready', enabled });
          } catch {
            if (!aborted) setState({ status: 'error', enabled: null });
          }
        })();
        return () => {
          aborted = true;
        };
      }, []);

      return state;
    }
    ```
