**### Recharts Usage Pattern**
Show the canonical code example for consuming tokens in recharts:

```tsx
// Primary series
<Line stroke="var(--accent)" strokeWidth={2} />

// Secondary series
<Line stroke="var(--chart-1)" strokeWidth={1.5} />
<Line stroke="var(--chart-2)" strokeWidth={1.5} />

// Area fill
<Area fill="var(--chart-highlight)" stroke="var(--accent)" />
```

Note: Recharts resolves CSS custom properties in SVG `stroke` and `fill` attributes on modern browsers. No JS color resolution needed.

**### Axis & Grid Conventions**
Per D-08: Axis tick labels use `--muted` color, 12px DM Sans:
```tsx
tick={{ fill: 'var(--muted)', fontSize: 12, fontFamily: 'DM Sans, sans-serif' }}
```
Per D-09: Cartesian grid lines use `--border-subtle` with dashed pattern:
```tsx
<CartesianGrid strokeDasharray="3 3" stroke="var(--border-subtle)" />
```
Per D-10: Clean charts (bar, radar) omit axis lines entirely. Grid lines only where they aid reading (line charts, area charts).

**### Tooltip Styling**
Per D-06 and D-07, all chart tooltips use this `contentStyle`:
```tsx
contentStyle={{
  backgroundColor: 'var(--surface)',
  border: '1px solid var(--border-subtle)',
  borderRadius: 'var(--radius-lg)',    // 8px
  boxShadow: '0 2px 8px rgba(0,0,0,0.08)',   // light
  fontFamily: 'DM Sans, sans-serif',
  fontSize: '13px',
  color: 'var(--ink)',
}}
```
Dark mode shadow: `0 2px 8px rgba(0,0,0,0.3)`. Note that CSS custom properties handle the color switch automatically; the shadow is the only value that does not auto-switch (acceptable — the difference is subtle).

**### Trajectory Language**
Per D-04, extend the athletic stat-line pattern from Readiness Signal Pattern:

Vocabulary (from strongest to weakest trajectory):
| Word | Meaning | When to Use |
|------|---------|-------------|
| ascending | Strong upward trend | slope > +3pts/session over 3+ sessions |
| climbing | Moderate improvement | slope +1 to +3pts/session |
| holding | Flat / stable | slope -1 to +1pts/session |
| dipping | Moderate decline | slope -1 to -3pts/session |
| stalling | Sustained weakness or decline | slope < -3pts/session or avg below threshold with no improvement |

Per D-05, two display formats:
- **Compact:** "[score] [trajectory word]" — e.g., "82 ascending" (used in roster badges, sparkline labels)
- **Narrative:** "Improving +Npts over M sessions" — e.g., "Improving +8pts over 3 sessions" (used in focus area hero, trend chart captions)

Narrative mapping:
| Trajectory | Narrative Prefix |
|-----------|-----------------|
| ascending / climbing | "Improving" |
| holding | "Holding steady" |
| dipping | "Slipping" |
| stalling | "Needs focus" |
