Note: TopBar is a client component. Since the associate layout is a server component, pass `role`, `associateSlug`, and `sidebarGroups` as serializable props. `associateSidebarGroups(slug)` returns plain objects with icon components — icon components (lucide-react) are function references. This means the sidebar groups must be constructed client-side. Create a thin client wrapper component `AssociateShellClient` that receives `slug` and `cohortName` as string props and renders TopBar + SectionSidebar with the correct config. Place it in `src/components/shell/AssociateShellClient.tsx`.

Actually, simpler approach: the layout can remain a server component, but since TopBar and SectionSidebar are both `'use client'` components, they can receive icon components as props from a server component in Next.js App Router — React Server Components can pass React component references to client components as part of the `children` or direct props. However, lucide icons are client components. The clean pattern is:

Make the associate layout a **client component** (`'use client'`) like the trainer dashboard layout is. Query the associate data server-side in a parent server layout or use a route segment. Actually, checking the trainer pattern: `trainer/layout.tsx` is a server component rendering `<TopBar />` (client), and `trainer/(dashboard)/layout.tsx` is `'use client'` rendering `<SectionSidebar>`.

Follow the same split pattern:
- `src/app/associate/[slug]/layout.tsx` — server component that fetches associate+cohort data, passes to a client shell
- Create `src/components/shell/AssociateShell.tsx` — `'use client'` component receiving `slug: string` and `cohortName: string | null`, renders TopBar + SectionSidebar + children layout

This way the server component handles data fetching and the client component handles interactive shell rendering.
