## Spacing Scale

Source: DESIGN.md

| Token | Value | Usage |
|-------|-------|-------|
| 2xs | 2px | Internal micro-gaps (icon inline) |
| xs | 4px | Icon gaps, badge padding, inline padding |
| sm | 8px | Compact element spacing, AvatarMenu gap |
| md | 16px | Default element spacing, TopBar horizontal padding |
| lg | 24px | Section padding, sidebar group margin-bottom |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions for this phase:
- Sidebar width (expanded): 200px (fixed — DESIGN.md Layout section)
- Sidebar width (collapsed): 48px (existing SectionSidebar behavior — no change)
- TopBar height: 56px (fixed — DESIGN.md + existing TopBar implementation)
- Sidebar nav item padding (expanded): `6px 12px` (existing SectionSidebar — no change)
- Sidebar nav item padding (collapsed): `8px 0` (existing SectionSidebar — no change)
- Sidebar nav item margin: `1px 6px` (existing SectionSidebar — no change)
- Cohort label padding: `0 12px 4px` top-of-sidebar header (matches group label pattern)
- Touch target minimum: 44px height for collapse toggle button

> **Grid exemption — SectionSidebar internal values:** The `6px`, `12px` (padding), and `1px` (margin) values above are locked to the existing `SectionSidebar` implementation and its companion `sidebar-configs.ts` styles. This phase does not modify `SectionSidebar` internals — it reuses the component as-is. These values are exempt from the 4-multiple grid rule for this phase. A separate refactor phase would be required to remap them to grid-aligned equivalents.

## Sidebar Header: Cohort Label

Condition: only rendered when associate has a cohort assigned (D-06, D-07).

Visual spec:
- Position: above the nav items, inside `<nav>` before the first group
- Content: `"Cohort: {cohortName}"` — literal prefix "Cohort: " followed by cohort display name
- Typography: 12px DM Sans 500, color `var(--muted)`, uppercase, letter-spacing 0.06em
- Padding: `0 12px 8px` (matches group label horizontal alignment, slightly more bottom gap than group labels)
- When no cohort: render nothing — no empty space, no placeholder (D-07)
- When collapsed: hide entirely (same rule as group labels in SectionSidebar)

## Accessibility Contract

- All nav items: `<Link>` with visible text (or `title` attribute when collapsed)
- Collapse toggle: `aria-label` declared (see Copywriting Contract)
- MobileSidebar: `aria-label="Associate navigation"` on mobile `<nav>`
- Focus-visible ring: 2px solid `var(--accent)`, offset 2px — applied consistently on all interactive elements
- Active nav item: color + left-border visual indicator (sufficient contrast — burnt orange on parchment exceeds 3:1 for UI components)
- No focus traps introduced by shell changes
- Sidebar collapse does not remove focused elements from the DOM — they remain keyboard-reachable when collapsed (title tooltip appears on hover/focus)
