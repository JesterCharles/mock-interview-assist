  <name>Task 1: Extend prisma/schema.prisma with Cohort, CurriculumWeek, and FK additions</name>
  <files>prisma/schema.prisma</files>
  <action>
Append two new models and modify Associate + Session per CONTEXT.md decisions D-05 through D-12.

1. Add `model Cohort`:
   - `id Int @id @default(autoincrement())`
   - `name String`
   - `startDate DateTime`
   - `endDate DateTime?`
   - `description String?`
   - `createdAt DateTime @default(now())`
   - `updatedAt DateTime @updatedAt`
   - `associates Associate[]`
   - `sessions Session[]`
   - `curriculumWeeks CurriculumWeek[]`

2. Add `model CurriculumWeek` (per D-07, D-08):
   - `id Int @id @default(autoincrement())`
   - `cohortId Int`
   - `weekNumber Int`
   - `skillName String`
   - `topicTags String[]` (Postgres native array)
   - `startDate DateTime`
   - `cohort Cohort @relation(fields: [cohortId], references: [id], onDelete: Cascade)`
   - `@@index([cohortId])`
   - NO unique constraint on (cohortId, weekNumber) — deferred to Phase 13 per D-09.

3. Modify `model Associate` (per D-02):
   - Add `cohortId Int?`
   - Add `cohort Cohort? @relation(fields: [cohortId], references: [id])`
   - Add `@@index([cohortId])` for roster filter queries (Phase 12 anticipates this).

4. Modify `model Session` (per D-10, D-12):
   - Add `cohortId Int?`
   - Add `cohort Cohort? @relation(fields: [cohortId], references: [id])`
   - Add `mode String @default("trainer-led")` (per D-10; valid values enforced in TS layer per D-11, no DB enum)
   - Add `@@index([cohortId])` for cohort-scoped session queries.

DO NOT use Prisma enums for `mode` (D-11 — matches existing pattern of no enums in this schema). DO NOT add `onDelete: Cascade` on Associate.cohort or Session.cohort — deleting a cohort should NOT delete associates or sessions; it should null the FK. Use Prisma default (`SetNull` semantics via nullable FK) — explicitly set `onDelete: SetNull` on both Associate.cohort and Session.cohort relations to be safe.
