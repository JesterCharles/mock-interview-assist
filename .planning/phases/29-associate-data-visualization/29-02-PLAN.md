interface SkillTrendChartProps {
  gapScores: GapScoreEntry[];
  sessions: SessionSummary[];

**Behavior per decisions:**

1. **Skill list (D-22):** Extract unique skills from gapScores. Build a dropdown in the chart header. When `selectedSkill` is non-null (dashboard filter active), auto-select that skill in the dropdown. Do NOT import/reuse the trainer's SkillFilterDropdown (it has hardcoded hex). Build inline or create a small styled select.

  <action>
Create `src/components/associate/SkillRadar.tsx` — `'use client'` component.

**Props interface:**
```typescript
interface SkillRadarProps {
  gapScores: GapScoreEntry[];        // Skill-level scores (pre-filtered by parent)
  selectedSkill: string | null;       // Dashboard filter — highlights one vertex
}
```

**Behavior per decisions:**

1. **Data preparation:** Filter gapScores to skill-level entries only (topic === '' or null). Map to radar data format:
   ```typescript
   { skill: string, score: number, assessmentReady: boolean }
   ```
   - score = `Math.round(weightedScore * 100)`
   - assessmentReady = `sessionCount >= 3` (D-12)

2. **Recharts RadarChart (D-12, D-13):**
   ```tsx
   <ResponsiveContainer width="100%" height={300}>
     <RadarChart data={radarData}>
       <PolarGrid stroke="var(--border-subtle)" />
       <PolarAngleAxis
         dataKey="skill"
         tick={({ payload, x, y, textAnchor, ...rest }) => {
           // Custom tick: assessment-ready = var(--ink), not ready = var(--muted) (D-12)
           const item = radarData.find(d => d.skill === payload.value);
           const isReady = item?.assessmentReady ?? false;
           return (
             <text
               x={x} y={y} textAnchor={textAnchor}
               fill={isReady ? 'var(--ink)' : 'var(--muted)'}
               fontSize={12}
               fontFamily="DM Sans, sans-serif"
               style={{ opacity: isReady ? 1 : 0.6 }}
             >
               {payload.value}
             </text>
           );
         }}
       />
       <Radar
         dataKey="score"
         stroke="var(--accent)"
         fill="var(--chart-highlight)"
         strokeWidth={2}
       />
     </RadarChart>
   </ResponsiveContainer>
   ```

3. **Assessment-ready distinction (D-12):** The Radar component draws a single polygon. For skills with <3 sessions, the line segments should appear dashed. Since recharts Radar doesn't support per-segment dash patterns natively, use this approach:
   - Render TWO Radar polygons overlaid: one for assessment-ready skills (solid, full opacity), one for all skills (dashed via `strokeDasharray="4 2"`, lower opacity `fillOpacity={0.1}`)
   - Or simpler: render a single Radar with dashed stroke, then overlay dots/markers for assessment-ready skills. Actually, the simplest correct approach: render the full polygon with `strokeDasharray="4 2"`, then render individual assessment-ready segments as solid overlays using `ReferenceDot` or custom shapes.
   
   **Simplest approach:** Render the full radar with a dashed stroke and muted fill. Then render individual dots at each vertex — assessment-ready vertices get solid `var(--accent)` dots (r=5), not-ready vertices get hollow dots with `var(--muted)` stroke. This visually communicates the distinction without fighting recharts' polygon rendering.

4. **Selected skill highlight (D-14):** When `selectedSkill` matches a vertex, render that dot larger (r=8) with `var(--accent)` fill and a subtle glow ring (use a second dot at r=12 with `var(--chart-highlight)` fill behind the main dot).

5. **Tooltip:** Use DESIGN.md tooltip contentStyle. Show skill name + score + "Assessment ready" or "Needs {3 - sessionCount} more sessions".

6. **Empty state:** If fewer than 3 skills, show a message "At least 3 skills needed for radar view" in `var(--muted)`.

7. **Imports from recharts:** `RadarChart, PolarGrid, PolarAngleAxis, PolarRadiusAxis, Radar, ResponsiveContainer, Tooltip`. All available in recharts 3.8.1.

8. **Card wrapper:** Background `var(--surface)`, border `1px solid var(--border-subtle)`, border-radius 12px, padding 16px. "Skill Overview" heading: Clash Display 18px 600.
