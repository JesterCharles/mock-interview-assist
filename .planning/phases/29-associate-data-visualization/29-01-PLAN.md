```typescript
export type TrendDirection = 'up' | 'down' | 'flat';

// D-04: Trend arrow direction from slope. up (>1), down (<-1), flat (-1 to 1)
export function getTrendDirection(slope: number): TrendDirection

// D-08, D-25: Narrative format "Improving +8pts over 3 sessions"
// Parameters: slope (for word), pointsDelta (signed int), sessionCount
export function getTrajectoryNarrative(slope: number, pointsDelta: number, sessionCount: number): string

// Compute per-skill trend slope from session summaries.
// Extracts scores for a specific skill from session assessments,
// computes linear regression slope over last N sessions.
// Returns -1 if fewer than 3 data points.
export function computeSkillTrend(
  sessions: Array<{ date: string; assessments: Record<string, { llmScore?: number; finalScore?: number }> }>,
  skill: string,
  gapScores: Array<{ skill: string; topic: string | null; weightedScore: number; sessionCount: number }>
): { slope: number; pointsDelta: number; sessionCount: number }
```

For `computeSkillTrend`: Since per-session per-skill scores aren't stored individually (GapScore is an aggregate), approximate the trend from the overall session scores filtered by whether the session included that skill. The function should:
1. Filter gapScores to skill-level entries (topic === '' or null) for the given skill
2. If sessionCount < 3, return { slope: -1, pointsDelta: 0, sessionCount }
3. Otherwise, use the session-level overall scores (sorted chronologically) to compute a linear regression slope
4. pointsDelta = most recent score - oldest score (from the available sessions)

Actually, simpler approach: the trend per skill should use the per-session overall technical scores since we don't have per-skill per-session breakdown. The slope represents general trajectory. For `pointsDelta`, compute from the weighted score change if possible, or from overall scores.

Simplest correct approach:
- `computeSkillTrend` takes the skill's current GapScoreEntry and the sessions array
- sessionCount from GapScoreEntry
- If sessionCount < 3, return slope=-1
- For slope: use overall session scores (last 3-5 sessions), compute linear regression
- For pointsDelta: difference between current weighted score and an approximation of the prior (use overall score delta as proxy)
- This is an approximation but matches what readinessService does

**Component behavior per decisions:**

1. **Data preparation:** Group gapScores by skill. Skill-level entries have `topic === '' || topic === null`. Topic-level entries have non-empty topic. For each skill, compute:

   - Score color via `getScoreColor(percent)` from vizUtils (D-15, D-16, D-18)
   - Trend direction via `getTrendDirection` and `computeSkillTrend` from vizUtils (D-04)
   - Topic breakdown: array of { topic, percent, color }

2. **Sort order (D-02):** Strongest first — sort by skill-level weightedScore descending.

3. **Card rendering (D-01):** Each skill card:
   - Collapsed state: skill name (left, DM Sans 14px 500, `var(--ink)`) + horizontal fill bar (flex-grow, height 8px, border-radius 4px, background `var(--border-subtle)`, fill width = percent%) + score percentage (right, DM Sans 14px 600, color matches bar per D-18) + trend arrow icon (lucide, 16px)
   - Trend arrows (D-04): `TrendingUp` from lucide with `color: var(--success)` for up, `TrendingDown` with `color: var(--danger)` for down, `Minus` with `color: var(--muted)` for flat
   - Chevron icon: `ChevronDown` / `ChevronUp` from lucide, `var(--muted)`, 16px — toggles expand/collapse

4. **Click behavior (D-03):** Clicking the card row does two things: (a) calls `onSelectSkill(skill)` to set dashboard filter (or null if already selected = toggle), (b) expands/collapses the card. Chevron icon click also toggles expand/collapse independently. Use `e.stopPropagation()` on chevron to prevent the row click handler from firing.

5. **Selected state (D-10):** When `selectedSkill === skill`, the card gets `border: 2px solid var(--accent)` and `background: var(--highlight)`. Other cards remain visible (highlight-only, nothing hides).

6. **Expanded state (D-01):** Shows topic breakdown below the skill bar. Each topic row:

   - Topic bars use same 5-band coloring as skill bars (D-16)
   - Animate expand/collapse: use CSS `max-height` transition or `grid-template-rows: 0fr → 1fr` pattern. Keep it simple — 200ms ease-out per DESIGN.md motion rules.

7. **Card styling:** Background `var(--surface)`, border `1px solid var(--border-subtle)`, border-radius `var(--radius-xl)` (12px), padding 16px. Gap between cards: 8px. When selected, border changes to `2px solid var(--accent)` (adjust padding by 1px to prevent layout shift).

8. **Empty state:** If no gapScores exist, render nothing (parent handles empty state).
