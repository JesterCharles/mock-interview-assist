```typescript
// GET /api/cohorts → CohortDTO[]
// POST /api/cohorts (CohortCreateInput) → CohortDTO (201)
// GET /api/cohorts/[id] → CohortDTO
// PATCH /api/cohorts/[id] (CohortUpdateInput) → CohortDTO
// DELETE /api/cohorts/[id] → 204

export interface CohortDTO {
  id: number;
  name: string;
  startDate: string;   // ISO
  endDate: string;     // ISO
  description: string | null;
  associateCount?: number;
}
```

Error shape: `{ error: string, issues?: ZodIssue[] }` with 400/401/404/409.

Fetch initial cohorts server-side with `prisma.cohort.findMany` (same shape as GET /api/cohorts) and pass to `<CohortsClient initialCohorts={...} />`.

    Create client component `CohortsClient.tsx` (per D-09, D-10, D-11, D-18):
    - State: `cohorts: CohortDTO[]`, `editingCohortId: number | null` (null = create mode when form open, -1 = closed, or use separate `formOpen: boolean`), `formState: { name, startDate, endDate, description }`, `formErrors: Record<string, string>`, `submitting: boolean`.
    - Render "New Cohort" button above table.
    - Table columns: Name, Start Date (format with `Intl.DateTimeFormat`), End Date, Associates (associateCount), Actions (Edit, Delete).
    - Inline form below header (or above table when open): fields for name, startDate (type="date"), endDate (type="date"), description (textarea). Submit button label changes to "Update" when editing. Cancel button clears form + closes.
    - On submit:
      - Create: POST /api/cohorts with `{ name, startDate, endDate, description: description || null }`. Handle 400 → map zod `issues` by path into `formErrors`. Handle 409 → show "A cohort with that name already exists" inline. On success, prepend to `cohorts`, close form.
      - Edit: PATCH /api/cohorts/[id] with same payload (partial accepted). Update matching row in state.
    - On Edit click: pre-fill form state with that row's values, set `editingCohortId = row.id`, open form.
    - On Delete click (per D-11): `if (!window.confirm(\`Delete cohort "\${name}"? Associates will be unassigned but kept.\`)) return;` → DELETE /api/cohorts/[id] → remove row from state on 204. Show inline error on failure.
    - All fetches use `credentials: 'include'` and `'Content-Type': 'application/json'`.

    Styling (per D-18): use DESIGN.md tokens. Reuse class patterns from `src/app/trainer/trainer.css` and `src/app/trainer/page.tsx`. Add a small `cohorts.css` for table + form specifics if the existing trainer.css doesn't cover it. Warm parchment background, Clash Display for the page heading ("Cohorts"), DM Sans body, burnt orange primary button.

    Inline error display: red text directly under each field with matching `formErrors[fieldName]`.

    Claude's discretion: exact markup/classes, whether to split `<CohortForm />` into its own file (recommended if the client file exceeds ~250 lines).
