# Forward-Looking Seeds from v1.3 Reflection

Seeds planted for future milestones. Each has a trigger condition that should surface it at the right time.

## Seed 1 — Real historical snapshots for radar Before/Now

**Trigger:** v1.4 planning, OR any milestone that touches associate data visualization.

**Context:** v1.3 SkillRadar component ships a Before/Now comparison using synthesized prior-state history. Labeled "Est. prior" with disclosure caption after codex flagged credibility risk, but the underlying data-fabrication pattern remains.

**Action:** Design a `SkillSnapshot` table (or use existing `GapScore` with a timestamp window) to store actual historical skill baselines. Replace synthesized prior with real snapshots computed from session data 30+ days before current.

---

## Seed 2 — Design-review precedes codex on shell/layout phases

**Trigger:** Any phase touching AppShell, TopBar, SectionSidebar, MobileSidebar, or top-level route layout.

**Context:** v1.3 Phase 32 + radar polish triggered 4 sequential codex passes, each finding 1-3 layout/visual issues. Visual iteration in codex is expensive (round-trips LLM calls on CSS).

**Action:** Add `/design-review` as a required pre-codex step in `/pipeline-execute` when phase touches shell components. Gate: no codex review submitted until design-review signoff.

---

## Seed 3 — Verify worktree-merged-to-main gate

**Trigger:** Every `/gsd-execute-phase` that uses superpowers worktrees.

**Context:** v1.3 Phase 32 plan-01 committed to a worktree branch (worktree-agent-a1486649) but was never merged to main. Verification reported `gaps_found` because main didn't have the changes. Subsequent plans had to redo the work.

**Action:** Add explicit step in `/gsd-execute-phase` that detects worktree branches and verifies they merged into main before running verification. Fail loudly if worktree commits are unmerged.

---

## Seed 4 — Demo/seed data namespace policy

**Trigger:** v1.4 planning, OR any new seed script, OR new `/api/admin/seed-*` route.

**Context:** v1.3 codex caught demo cohort prefix collision with real trainer-created cohorts. Fix: prefix all demo data with `demo_` or `_demo_`. Policy needs to be documented, not rediscovered per-script.

**Action:** Add "Demo/Seed Data Namespacing" section to PROJECT.md. Create `src/lib/demoNamespace.ts` helper that prefixes + validates demo identifiers. Enforce via lint rule on seed scripts.

---

## Seed 5 — Client-side API contract tests

**Trigger:** Any UI component that makes `fetch`/POST calls to internal API routes.

**Context:** v1.3 cohort-switcher shipped with query param mismatch vs API route contract; filter didn't persist across navigation. Fix: 5e0ebdb. Pattern: testing the API route in isolation missed the client-side contract drift.

**Action:** Add Vitest component-level tests that mock `fetch` and assert the request URL + body matches API route's zod schema. At minimum for: CohortSwitcher, profile form submit, setup wizard submit.

---

## Seed 6 — Auto-trigger /cso on auth-adjacent mid-execute scope growth

**Trigger:** Any phase that adds/modifies code under `src/lib/auth-*`, `src/app/api/auth/*`, or `src/middleware.ts`.

**Context:** v1.3 Phase 28.1 (user profile, bonus mid-execute scope) shipped `passwordSetAt` writable via PUT /api/profile (CRITICAL). Caught by codex, not by /cso, because /cso wasn't run on the bonus scope.

**Action:** Lint rule or pre-commit hook: if diff touches auth-adjacent paths AND current phase wasn't planned up-front (i.e., it's a `.1` or bonus phase), require `/cso` to run before PR merge.
