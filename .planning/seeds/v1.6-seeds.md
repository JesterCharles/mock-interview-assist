## Carryovers from v1.5 Retro Action Items

Copied verbatim from `.planning/PIPELINE-REFLECT.md` § Action Items for v1.6 (origin marker `retro-v1.5` applied to each).

1. **[retro-v1.5] Judge0 self-hosted IaC + `terraform apply` on prod.** v1.5 shipped Judge0 flag-dark behind `isCodingEnabled()`. v1.6 owns live Judge0 provisioning + coding-challenges enablement. Starting point: `iac/gce-judge0/` reference template (Phase 50 D-13) + Phase 38 spike learnings.
2. **[retro-v1.5] Real-Postgres SQL runner.** v1.4 SQL shipped as SQLite-via-Judge0 (MVP). v1.6 needs a real Postgres runner (hardened, isolated, per-attempt teardown).
3. **[retro-v1.5] Drop the no-auto-merge policy.** Phase 48 landed CI/CD workflows. Once staging + prod deploys have soaked for a sprint, flip merge policy from "human only" to "CI-gated auto-merge on approved PRs."
4. **[retro-v1.5] Redis-backed rate-limit store.** File-backed `rate-limits.json` does not survive Cloud Run horizontal scale. Migrate to Memorystore / Upstash / Supabase Edge KV.
5. **[retro-v1.5] Operator-gate batching UX.** Introduce a "live-ops window" concept at pipeline-plan time — rather than per-plan halts, batch live-infra operator steps into named 1-2h windows. v1.5 accumulated 20+ queued gates; v1.6 should not repeat.
6. **[retro-v1.5] Resume + complete the v1.5 live infra queue.** Not v1.6 per se, but the bridge: operator runs the 20+ queued live steps from P45-52, then `/pipeline-review` (codex) + `/pipeline-test` (staging soak + abuse artifact) + `/pipeline-ship` (DNS cutover + tag push) before v1.6 discover starts.
7. **[retro-v1.5] Dependency-aware executor ordering.** v1.4 Phase 42 + v1.5 Phase 50-02 ∩ Phase 48 both hit "numeric order ≠ dependency order" pain. Build minimal topological ordering into `gsd-tools` executor.
8. **[retro-v1.5] Autonomous-executor `package.json` write hook.** P44 CR-01 class of bugs: agent authors script using a new dep but forgets package.json. Auto-sync via hook.
9. **[retro-v1.5] Language-ID map as live source-of-truth query.** Eliminate the JUDGE0_LANGUAGE_MAP drift class by querying `/languages` at boot and caching rather than hard-coding IDs.
10. **[retro-v1.5] Lint warning backlog cleanup.** 183 project warnings carried from v1.0 era. One dedicated 2h plan in v1.6 clears them. Low priority but easy win.
11. **[retro-v1.5] Modern .NET 8+ runtime** on Judge0 (pending client demand validation).
12. **[retro-v1.5] In-app challenge authoring UX.** PR-based authoring works but limits trainer velocity.
13. **[retro-v1.5] Multi-region HA / failover.** Single-region Cloud Run is MVP-acceptable but adds a single-point-of-failure at scale.
14. **[retro-v1.5] Observability stack beyond Cloud Run built-ins.** Phase 48 shipped native Cloud Run monitoring; Grafana/Prom + on-call pager are v1.6+.
15. **[retro-v1.5] 999.2 Trainer Default Cohort.** v1.1 backlog item — trainers typically own one cohort; "All Cohorts" default is noise.
