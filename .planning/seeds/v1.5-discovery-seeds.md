# v1.5 Discovery Seeds

Forward-looking ideas captured during v1.5 discover. Trigger conditions indicate when each seed should resurface.

---

## Seed 1: AWS App Migration (Supabase Stays)

**Idea:** Migrate the **app layer only** from Cloud Run to AWS (ECS Fargate or App Runner). **Supabase stays unchanged** — it's company-standard, cloud-agnostic, and serves other company projects already. Only the app runtime + GH Actions deploy step changes.

**Trigger:** (a) GCP free credits exhausted OR (b) sales/pitch unlocks company-aligned infra budget OR (c) company mandate aligns NLM with existing AWS footprint.

**Preserves:** Dockerfile unchanged. Supabase DB + auth unchanged (no Cognito/RDS migration). `DATABASE_URL` + `NEXT_PUBLIC_SUPABASE_URL` unchanged. Only app runtime + CI/CD deploy target changes.

**Effort estimate:** S-M (1-2 weeks). No auth or DB migration risk.

---

## Seed 2: Observability Stack (Self-Hosted Metrics)

**Idea:** Prometheus + Grafana + Loki stack for app metrics + logs + traces. Cloud Run built-ins are fine for v1.5 MVP but not enough for prod debugging at scale.

**Trigger:** First incident where Cloud Run logs aren't enough to root-cause a prod issue OR concurrent user count exceeds the load-test ceiling.

**Approach:** Prometheus scrape endpoint on the app (already planned in v1.5 infra table). Grafana Cloud free tier first; self-host on a GCE micro-VM only if free tier insufficient.

---

## Seed 3: Blue/Green Deploy + Feature Flags at Request Level

**Idea:** Move past tag-based deploys to Cloud Run traffic-splitting (0% → 10% → 100%) + LaunchDarkly / PostHog feature flags for per-user rollout.

**Trigger:** First v1.5 rollback OR first time a feature needs to go live for 10% of users before full launch OR coding challenges go live in v1.6 (risk).

**Cost:** LaunchDarkly free tier = 1000 MAU. PostHog self-hosted = free.

---

## Seed 4: Think-Out-Loud + STT + Agent Code-Explainability Feature

**Idea (from user):** Future feature where associates think out loud during coding challenges. STT captures their reasoning. An agent evaluates the thought process + code explainability (not just correctness).

**Trigger:** v1.6 Judge0 live + coding challenges proven working across all 5 languages (Python, JS/TS, Java, SQL, C#) + rollout plan hitting real users.

**Dependencies:** Web Speech API (already used for mock interviews), OpenAI Whisper for offline STT fallback, LangGraph workflow extension.

**Milestone target:** v1.7 or v1.8.

---

## Seed 5: Cost-per-User Dashboard (for pitch)

**Idea:** Surface the load-test output as a live dashboard. "At current traffic you could support N concurrent users for $M/month." Derived from k6 baseline + Cloud Run billing API.

**Trigger:** Load test produces first baseline AND pitch walk-through includes a scaling/economics question.

**Approach:** Static page auto-updated from GitHub Actions that re-runs k6 weekly + hits Cloud Run pricing API. 1-2 day build.

---

## Seed 6: Supabase Capacity / Tier Review

**Idea:** Review Supabase plan tier + feature utilization as usage grows. Supabase is permanent (company-standard), so "do we leave Supabase" is NOT the question. "Are we on the right Supabase tier for our load" is.

**Trigger:** (a) Free tier / Pro tier limits hit (DB size, MAU, bandwidth, edge function invocations) OR (b) k6 load test shows Supabase connection pool saturation before app saturation OR (c) compliance need (HIPAA / SOC2 / data residency) requires Enterprise tier.

**Do NOT trigger on:** "We should eventually own our DB." That was rejected at v1.5 kickoff. Supabase stays.

---

## Seed 7: Judge0.com Hosted (Backup Path)

**Idea:** If v1.6 Judge0 self-hosted turns out to be more ops tax than expected, pivot to Judge0.com hosted API ($0 dev → $20/100k subs).

**Trigger:** v1.6 Judge0 VM setup takes >1 week OR first Judge0 incident requires >4h of debugging OR Judge0.com pricing drops.

**Break-even reminder:** ~150k submissions/month. Below that, hosted wins on ops-tax math.

---

## Seed 8: Demo-Data Seeder

**Idea:** `npm run seed:demo` script that populates a fresh DB with realistic cohorts, associates, sessions, gap scores. For champion pitch walk-through.

**Trigger:** Before v1.5 pitch + each time demo env gets stale.

**Dependencies:** Extend existing `prisma/seed.ts` OR new `scripts/seed-demo.ts`. 1-day build.

---

## Seed 9: CI/CD Branch Protection + Required Checks

**Idea:** Enforce `main` branch protection: require PR, require `pr-checks.yml` pass, require Codex review approval, require 1 human review. Matches solo-dev cadence today but prevents accidents later.

**Trigger:** v1.5 ship OR first teammate joins OR first accidental push-to-main.

**Effort:** 30 min (GitHub settings + one-time).
