<feature>
  <name>Recency-Weighted Gap Scoring Algorithm</name>
  <files>src/lib/gapService.ts, src/lib/__tests__/gapService.test.ts</files>
  <behavior>
    ## Core Algorithm (D-01, GAP-02)
    - recencyWeightedAverage([80]) = 80 (single score, weight 1.0)
    - recencyWeightedAverage([80, 60]) = (80*1.0 + 60*0.8) / (1.0+0.8) = 71.11 (newer session weighted more)
    - recencyWeightedAverage([90, 70, 50]) = (90*1.0 + 70*0.8 + 50*0.64) / (1.0+0.8+0.64) = 73.77
    - recencyWeightedAverage([]) = 0 (empty array edge case)

    ## Score Extraction (D-03)
    - extractScore({finalScore: 85, llmScore: 70, didNotGetTo: false}) = 85 (prefer finalScore)
    - extractScore({llmScore: 70, didNotGetTo: false}) = 70 (fallback to llmScore)
    - extractScore({didNotGetTo: true, finalScore: 85}) = null (skipped, ignore)
    - extractScore({didNotGetTo: false}) = null (no score available)

    ## Two-Level Gap Computation (GAP-01, D-02)
    - Given a session with techMap {1: "react"} and questions from week 1 with keywords ["hooks", "state"]:
      - Produces skill-level score for "react" (topic="")
      - Produces topic-level scores for "react"/"hooks" and "react"/"state"
    - Starter questions (id prefix "starter-") are excluded (Pitfall 4)
    - Questions with didNotGetTo=true are excluded (Pitfall 3)

    ## Keyword Normalization (GAP-04)
    - Keywords normalized: trimmed, lowercased
    - "React Hooks" -> "react hooks", " useState " -> "usestate"

    ## Multi-Session Aggregation
    - Given 3 sessions for an associate, computes per-skill weighted averages across sessions (newest first)
    - Each session contributes one average score per skill (mean of that session's question scores for that skill)
  </behavior>
  <implementation>
After tests are written and failing (RED), implement in `src/lib/gapService.ts`:

1. **recencyWeightedAverage(scores: number[]): number** — Pure function. Scores ordered newest-first. Apply 0.8^index decay. Return normalized weighted average. Return 0 for empty array.

2. **extractScore(assessment: QuestionAssessment): number | null** — Return finalScore if defined, else llmScore if defined. Return null if didNotGetTo is true or neither score exists.

3. **extractSkillTopicScores(session: InterviewSession): Map<string, Map<string, number[]>>** — Parse a single session's assessments into a nested map: skill -> topic -> scores[]. Use techMap to resolve weekNumber to skill name. Filter out starter questions and skipped questions. Normalize keywords to lowercase/trimmed. Also compute skill-level aggregate (topic key = "").

4. **computeGapScores(sessions: InterviewSession[]): GapScoreInput[]** — Takes multiple sessions (newest first). For each skill and topic, collect per-session average scores, then apply recencyWeightedAverage. Return array of {skill, topic, weightedScore, sessionCount} objects ready for Prisma upsert.

Type for the output:
```typescript
export interface GapScoreInput {
  skill: string;
  topic: string;     // "" for skill-level
  weightedScore: number;
  sessionCount: number;
}
```
