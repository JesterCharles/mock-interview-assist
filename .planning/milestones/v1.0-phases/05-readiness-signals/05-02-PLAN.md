    **Part A: settingsService.ts**

    Create src/lib/settingsService.ts:

    ```typescript
    import prisma from '@/lib/prisma';
    import { recomputeAllReadiness } from '@/lib/readinessService';

    export interface AppSettings {
      readinessThreshold: number;
    }

    export async function getSettings(): Promise<AppSettings> {
      const settings = await prisma.settings.findFirst({ where: { id: 1 } });
      return {
        readinessThreshold: settings?.readinessThreshold ?? 75,
      };
    }

    export async function updateThreshold(newThreshold: number): Promise<void> {
      await prisma.settings.upsert({
        where: { id: 1 },
        create: { id: 1, readinessThreshold: newThreshold },
        update: { readinessThreshold: newThreshold },
      });
      // Bulk recompute all associates with new threshold
      // Synchronous batch — safe for MVP (<200 associates).
      // If scale grows beyond ~200, move to background task.
      await recomputeAllReadiness(newThreshold);
    }
    ```

    **Part B: API route**

    Create src/app/api/settings/route.ts:

    - Import `isAuthenticatedSession` from `@/lib/auth-server`
    - Import `getSettings`, `updateThreshold` from `@/lib/settingsService`
    - Import `z` from `zod` (version 4.3.6 per CLAUDE.md)

    **GET handler:**
    1. Check `isAuthenticatedSession()` — return 401 JSON if false
    2. Call `getSettings()` and return as JSON

    **PUT handler:**
    1. Check `isAuthenticatedSession()` — return 401 JSON if false
    2. Parse body as JSON
    3. Validate with zod: `z.object({ readinessThreshold: z.number().min(0).max(100) })`
    4. If validation fails, return 400 with error message
    5. Call `updateThreshold(parsed.readinessThreshold)`
    6. Return 200 JSON `{ success: true }`

    Wrap PUT handler body in try/catch. If zod parse throws, return 400 with the zod error message. If any other error, return 500.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Spoofing | /api/settings GET+PUT | mitigate | Guard both handlers with `isAuthenticatedSession()` — reject with 401 if unauthenticated. Same pattern as all protected routes in this app. |
| T-05-05 | Tampering | PUT /api/settings body | mitigate | Validate with `zod z.number().min(0).max(100)` at route entry. Reject NaN, Infinity, negative, >100 with 400. |
| T-05-06 | Denial of Service | recomputeAllReadiness via PUT | mitigate | Auth guard prevents unauthenticated callers from triggering bulk recompute. Rate limiting already exists on the app (rateLimitService.ts). For MVP scale, synchronous batch is acceptable. |
