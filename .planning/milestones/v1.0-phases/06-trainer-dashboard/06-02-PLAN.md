<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> /api/trainer/[slug] | User-controlled slug parameter in URL |
| client -> recharts | User-viewable data rendered in SVG (no injection risk — React JSX escapes) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Tampering / Injection | /api/trainer/[slug] slug param | mitigate | Validate slug matches `/^[a-z0-9-]+$/` before using in Prisma query. Return 400 for invalid slugs. Prisma parameterized queries prevent SQL injection regardless, but input validation is defense-in-depth. |
| T-06-05 | Information Disclosure | /api/trainer/[slug] | mitigate | Auth check via nlm_session cookie before returning any associate data. Return 401 for unauthenticated requests. |
| T-06-06 | Tampering / XSS | Chart data rendering | accept | React JSX escapes all interpolated values by default. No `dangerouslySetInnerHTML` used. recharts renders SVG which is also safe against injection. Low risk. |
| T-06-07 | Denial of Service | /api/trainer/[slug] large session query | accept | Query limited to `take: 10` sessions and gapScores are bounded by skill count. No unbounded queries. Acceptable for single-user MVP. |
</threat_model>
