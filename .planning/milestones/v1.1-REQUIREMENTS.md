## Goal

Build a trusted readiness record that any evidence source can feed — starting with connecting automated interviews to the readiness pipeline, adding associate PIN-based auth, cohort management, and curriculum-driven question selection.

**Final scope:** 14 requirements across 5 categories. DESIGN-03 was added mid-milestone (Phase 15 scope expansion — overrode Codex #8's legacy-preservation intent).

## Completed Requirements (v1.1)

### Auth (PIN-based, lightweight)

- [x] **AUTH-01**: Trainer generates a 6-digit PIN when adding an associate to a cohort. PIN is unique per associate and displayed to trainer for manual communication.
  - **Outcome:** Shipped via `/api/associate/pin/generate` + `GeneratePinButton`. bcryptjs-hashed, shown once. **Feature-gated off (`ENABLE_ASSOCIATE_AUTH=false`) for v1.1 ship** per Codex P1.
- [x] **AUTH-02**: Associate enters PIN at start of automated interview. System validates PIN and creates an associate session (HttpOnly cookie) that links the interview to their identity.
  - **Outcome:** Shipped via `/api/associate/pin/verify` + `associateSession.ts`. Cookie HMAC-signed with dedicated `ASSOCIATE_SESSION_SECRET`; version = `pinGeneratedAt` for DB-backed revocation.
- [x] **AUTH-03**: Associate auth coexists with trainer auth. Middleware uses identity enumeration (`trainer | associate | anonymous`) with separate cookie names. No cross-contamination.
  - **Outcome:** Cookie-only identity enum in `src/middleware.ts:29-58` (Edge-safe; no DB). Trainer gate rejects associate via redirect to `/login`.
- [x] **AUTH-04**: Associate profile page `/associate/[slug]` requires auth. Either matching associate session OR active trainer session grants access. Mismatched associates return 403. Unauthenticated → PIN entry redirect.
  - **Outcome:** Shipped via `getAssociateSession` + JSX-wrapped 403 shell (Next.js page components cannot return raw Response).

### Pipeline Integration

- [x] **PIPE-01**: Automated interview sessions link to the authenticated associate via PIN session. No associate slug input required — identity from session.
  - **Outcome:** Shipped via `/api/associate/interview/complete` which derives identity from cookie only; client-supplied `associateSlug` is ignored. Codex #3 closure.
- [x] **PIPE-02**: Automated interview sessions trigger gap score computation and readiness classification update on completion, using the same pipeline as trainer-led sessions.
  - **Outcome:** Shared `runReadinessPipeline` + `Session.readinessRecomputeStatus` marker + `/api/admin/readiness-sweep` repair endpoint (Codex #5).

### Cohort Management

- [x] **COHORT-01**: Trainer can create, edit, and delete cohorts via trainer dashboard. Cohort fields: name, start date, end date, description.
  - **Outcome:** `/api/cohorts` + `/api/cohorts/[id]` CRUD with zod validation; non-cascading DELETE via `prisma.$transaction`. UI at `/trainer/cohorts`.
- [x] **COHORT-02**: Trainer can assign an associate to one cohort via nullable FK. Associates without a cohort remain functional (backward compatible with v1.0).
  - **Outcome:** `PATCH /api/trainer/[slug]` + `AssociateCohortSelect` dropdown. `schema.prisma`: `cohortId Int?` + `onDelete: SetNull`.
- [x] **COHORT-03**: Trainer dashboard roster page supports filtering by cohort via dropdown selector. Roster table and readiness badges unchanged.
  - **Outcome:** `/api/trainer?cohortId=` filter + `CohortFilter` component. Default response shape unchanged (Codex #1).
- [x] **COHORT-04**: Cohort view displays aggregate readiness summary: count of ready / improving / not_ready. Summary bar above roster.
  - **Outcome:** `ReadinessSummaryBar` + `?includeSummary=true` opt-in wrapped response. No v1.0 consumer breaks.

### Curriculum

- [x] **CURRIC-01**: Trainer can define curriculum weeks per cohort. `CurriculumWeek` fields: cohort FK, week number, skill name, **skillSlug (canonical)**, topic tags, start date.
  - **Outcome:** `/api/cohorts/[id]/curriculum` CRUD + `/trainer/cohorts/[id]/curriculum` UI. `skillSlug` regex-validated; `@@unique([cohortId, weekNumber])` at DB level; `409` on unique violation (Codex #9).
- [x] **CURRIC-02**: Interview setup auto-filters available questions to skills from taught curriculum weeks (where `startDate <= today`). Adaptive weights compose on top.
  - **Outcome:** `filterTechsByCurriculum` uses `Set.has(firstSegment)` for **exact** skillSlug match (`react` ≠ `react-native`). Wizard separates `allTechs` source from `availableTechs` visible state so filter never compounds (Codex P2 fix). `filterGapScoresByCurriculum` runs before `applyGapScores`.

### Design

- [x] **DESIGN-01**: Public interview flow (landing, PIN entry, automated interview), associate profile page, and auth pages styled per DESIGN.md tokens.

- [x] **DESIGN-02**: New cohort management UI and curriculum UI follow DESIGN.md from initial build. No retrofit debt.
  - **Outcome:** Zero grep hits for legacy utilities in `src/components/cohort`, `src/components/curriculum`, `src/app/trainer/cohorts`.
- [x] **DESIGN-03** (added mid-milestone): All routes unified; legacy `--nlm-*` tokens + utilities (`.glass-card`, `.gradient-text`, `.nlm-bg`, glow classes, motion keyframes) deleted from `globals.css`.
  - **Outcome:** 50 → 0 legacy refs across `src/`. `globals.css` collapsed from ~600 → 179 lines. Unified `/signin` tabs replace split logins. 24/24 Playwright green. Atomic deletion commit `06987c7`. Overrides Codex #8's legacy-preservation intent.

## Out of Scope (v1.1 and likely beyond)

- Associate self-registration / public signup — trainer adds associates
- Cohort invitation emails with onboarding — PIN delivery is trainer-mediated
- Multi-trainer / role-based access within a cohort
- Automated interview scheduling / calendar integration — curriculum `startDate` sufficient
- Cohort progress report PDFs — no client-facing portal yet
- Real-time dashboard updates — polling sufficient
- Multi-cohort per associate / specialization tracks — single FK sufficient
