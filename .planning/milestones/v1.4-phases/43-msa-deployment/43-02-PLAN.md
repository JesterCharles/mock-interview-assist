Create `.github/workflows/deploy-app.yml` per D-07 + IAC-03. Triggers on tag push matching `app-v*`. Builds → pushes to GCR → SSH-deploys → probes `/api/health` → rollback on failure.

```yaml
name: Deploy App

# -----------------------------------------------------------------------------
# Required GitHub Actions secrets (Settings → Secrets and variables → Actions):
#   GCP_SA_KEY       service account JSON (storage.admin + compute.instanceAdmin)
#   GCP_PROJECT_ID   GCP project id (e.g. "nlm-prod")
#   APP_VM_NAME      name of app VM (matches Terraform app_vm_name)
#   APP_VM_ZONE      app VM zone (e.g. "us-central1-a")
#   APP_VM_SSH_KEY   private SSH key authorized on app VM via OS Login
#
# Trigger: push a git tag `app-v<SEMVER>` (e.g. git tag app-v1.4.0 && git push --tags)
# Image tag mapping: app-v1.4.0 -> gcr.io/$PROJECT/app:1.4.0
# -----------------------------------------------------------------------------

on:
  push:
    tags:
      - 'app-v*'

concurrency:
  group: deploy-app-prod
  cancel-in-progress: false

jobs:
  deploy:
    runs-on: ubuntu-latest
    timeout-minutes: 20
    permissions:
      contents: read
      id-token: write
    steps:
      - uses: actions/checkout@v4

      - name: Extract image tag from git tag
        id: tag
        run: |
          IMAGE_TAG="${GITHUB_REF_NAME#app-v}"
          echo "image_tag=$IMAGE_TAG" >> $GITHUB_OUTPUT
          echo "full_image=gcr.io/${{ secrets.GCP_PROJECT_ID }}/app:$IMAGE_TAG" >> $GITHUB_OUTPUT

      - name: Auth to GCP
        uses: google-github-actions/auth@v2
        with:
          credentials_json: ${{ secrets.GCP_SA_KEY }}

      - name: Setup gcloud
        uses: google-github-actions/setup-gcloud@v2

      - name: Configure Docker for GCR
        run: gcloud auth configure-docker --quiet

      - name: Build and push image
        run: |
          docker build -t ${{ steps.tag.outputs.full_image }} .
          docker tag ${{ steps.tag.outputs.full_image }} gcr.io/${{ secrets.GCP_PROJECT_ID }}/app:latest
          docker push ${{ steps.tag.outputs.full_image }}
          docker push gcr.io/${{ secrets.GCP_PROJECT_ID }}/app:latest

      - name: Setup SSH key
        run: |
          mkdir -p ~/.ssh
          echo "${{ secrets.APP_VM_SSH_KEY }}" > ~/.ssh/id_ed25519
          chmod 600 ~/.ssh/id_ed25519

      - name: Capture previous image for rollback
        id: prev
        run: |
          PREV_TAG=$(gcloud compute ssh ${{ secrets.APP_VM_NAME }} \
            --zone=${{ secrets.APP_VM_ZONE }} \
            --ssh-key-file=~/.ssh/id_ed25519 \
            --command="docker inspect --format='{{index .Config.Image}}' interview-assistant 2>/dev/null || echo gcr.io/${{ secrets.GCP_PROJECT_ID }}/app:latest" \
            --quiet 2>/dev/null || echo "gcr.io/${{ secrets.GCP_PROJECT_ID }}/app:latest")
          echo "prev_image=$PREV_TAG" >> $GITHUB_OUTPUT
          echo "Previous image recorded for rollback: $PREV_TAG"

      - name: Deploy new image to app VM
        id: deploy
        run: |
          gcloud compute ssh ${{ secrets.APP_VM_NAME }} \
            --zone=${{ secrets.APP_VM_ZONE }} \
            --ssh-key-file=~/.ssh/id_ed25519 \
            --command="set -e; cd ~/app && gcloud auth configure-docker --quiet && docker pull ${{ steps.tag.outputs.full_image }} && docker tag ${{ steps.tag.outputs.full_image }} gcr.io/${{ secrets.GCP_PROJECT_ID }}/app:latest && docker compose up -d --no-deps interview-assistant"

      - name: Health check
        id: health
        run: |
          sleep 20
          for i in 1 2 3 4 5; do
            STATUS=$(gcloud compute ssh ${{ secrets.APP_VM_NAME }} \
              --zone=${{ secrets.APP_VM_ZONE }} \
              --ssh-key-file=~/.ssh/id_ed25519 \
              --command="curl -s -o /dev/null -w '%{http_code}' http://localhost:3000/api/health" \
              --quiet 2>/dev/null || echo "000")
            echo "Health check attempt $i: HTTP $STATUS"
            if [ "$STATUS" = "200" ]; then
              echo "health=ok" >> $GITHUB_OUTPUT
              exit 0
            fi
            sleep 10
          done
          echo "health=fail" >> $GITHUB_OUTPUT
          exit 1

      - name: Rollback on health failure
        if: failure() && steps.health.outputs.health == 'fail'
        run: |
          echo "::error::Health check failed. Rolling back to ${{ steps.prev.outputs.prev_image }}"
          gcloud compute ssh ${{ secrets.APP_VM_NAME }} \
            --zone=${{ secrets.APP_VM_ZONE }} \
            --ssh-key-file=~/.ssh/id_ed25519 \
            --command="set -e; cd ~/app && docker pull ${{ steps.prev.outputs.prev_image }} && docker tag ${{ steps.prev.outputs.prev_image }} gcr.io/${{ secrets.GCP_PROJECT_ID }}/app:latest && docker compose up -d --no-deps interview-assistant"
          echo "::error::Rollback complete. Deploy FAILED — investigate before retrying."
          exit 1
```

Notes for executor:
- Concurrency group `deploy-app-prod` + `cancel-in-progress: false` = serialized deploys, never cancel mid-flight
- Image tag convention per D-07: strip `app-v` prefix
- Rollback per D-07: SSH + `docker pull` previous tag + `docker compose up -d --no-deps interview-assistant`
- All SSH via `gcloud compute ssh` (OS Login, IAM-respecting)
- Health check: 5 retries × 10s backoff = ~70s window
- GitHub Actions auto-masks secret values; do NOT use `set -x` in run blocks

  <files>.github/workflows/deploy-judge0.yml</files>
  <action>
Create `.github/workflows/deploy-judge0.yml` per D-08 + IAC-03. Triggers on tag push matching `judge0-v*`. SSHes to Judge0 VM, pulls pinned Judge0 image, `docker compose up -d`, probes `/system_info`, rolls back on failure.

Important: Judge0 image version lives in the `docker-compose.yml` ON THE JUDGE0 VM (not the app repo). The Judge0 compose file is version-controlled separately on the VM at `~/judge0/docker-compose.yml`. This workflow:
1. Records the current Judge0 compose file (backup copy for rollback)
2. Overwrites the compose file's `JUDGE0_IMAGE_TAG` env var with the tag from the git ref
3. Runs `docker compose pull && docker compose up -d`
4. Probes `/system_info` via curl with `X-Auth-Token` header
5. On failure, restores the backup compose file and re-runs `up -d`

```yaml
name: Deploy Judge0

# -----------------------------------------------------------------------------
# Required GitHub Actions secrets (Settings → Secrets and variables → Actions):
#   GCP_SA_KEY         service account JSON (compute.instanceAdmin on Judge0 VM)
#   GCP_PROJECT_ID     GCP project id
#   JUDGE0_VM_NAME     name of Judge0 VM (default from Plan 43-01: "nlm-judge0-prod")
#   JUDGE0_VM_ZONE     Judge0 VM zone
#   JUDGE0_VM_SSH_KEY  private SSH key authorized on Judge0 VM via OS Login
#   JUDGE0_AUTH_TOKEN  X-Auth-Token configured in Judge0 server (Phase 38 JUDGE-04)
#
# Trigger: push a git tag `judge0-v<VERSION>` (e.g. git tag judge0-v1.13.1 && git push --tags)
# Image tag mapping: judge0-v1.13.1 -> judge0/api:1.13.1
#
# Assumes the Judge0 compose file on the VM is ~/judge0/docker-compose.yml and
# references the image via ${JUDGE0_IMAGE_TAG} env var (set in ~/judge0/.env).
# -----------------------------------------------------------------------------

on:
  push:
    tags:
      - 'judge0-v*'

concurrency:
  group: deploy-judge0-prod
  cancel-in-progress: false

jobs:
  deploy:
    runs-on: ubuntu-latest
    timeout-minutes: 20
    permissions:
      contents: read
    steps:
      - uses: actions/checkout@v4

      - name: Extract Judge0 version from git tag
        id: tag
        run: |
          JUDGE0_VERSION="${GITHUB_REF_NAME#judge0-v}"
          echo "judge0_version=$JUDGE0_VERSION" >> $GITHUB_OUTPUT
          echo "Deploying Judge0 version: $JUDGE0_VERSION"

      - name: Auth to GCP
        uses: google-github-actions/auth@v2
        with:
          credentials_json: ${{ secrets.GCP_SA_KEY }}

      - name: Setup gcloud
        uses: google-github-actions/setup-gcloud@v2

      - name: Setup SSH key
        run: |
          mkdir -p ~/.ssh
          echo "${{ secrets.JUDGE0_VM_SSH_KEY }}" > ~/.ssh/id_ed25519
          chmod 600 ~/.ssh/id_ed25519

      - name: Backup current Judge0 config for rollback
        id: prev
        run: |
          PREV_VERSION=$(gcloud compute ssh ${{ secrets.JUDGE0_VM_NAME }} \
            --zone=${{ secrets.JUDGE0_VM_ZONE }} \
            --ssh-key-file=~/.ssh/id_ed25519 \
            --command="cd ~/judge0 && cp .env .env.rollback && grep JUDGE0_IMAGE_TAG .env | cut -d= -f2" \
            --quiet 2>/dev/null || echo "1.13.1")
          echo "prev_version=$PREV_VERSION" >> $GITHUB_OUTPUT
          echo "Previous Judge0 version recorded for rollback: $PREV_VERSION"

      - name: Update Judge0 image tag and deploy
        id: deploy
        run: |
          gcloud compute ssh ${{ secrets.JUDGE0_VM_NAME }} \
            --zone=${{ secrets.JUDGE0_VM_ZONE }} \
            --ssh-key-file=~/.ssh/id_ed25519 \
            --command="set -e; cd ~/judge0 && sed -i 's/^JUDGE0_IMAGE_TAG=.*/JUDGE0_IMAGE_TAG=${{ steps.tag.outputs.judge0_version }}/' .env && docker compose pull && docker compose up -d"

      - name: Health check /system_info
        id: health
        run: |
          sleep 30
          for i in 1 2 3 4 5; do
            STATUS=$(gcloud compute ssh ${{ secrets.JUDGE0_VM_NAME }} \
              --zone=${{ secrets.JUDGE0_VM_ZONE }} \
              --ssh-key-file=~/.ssh/id_ed25519 \
              --command="curl -s -o /dev/null -w '%{http_code}' -H 'X-Auth-Token: ${{ secrets.JUDGE0_AUTH_TOKEN }}' http://localhost:2358/system_info" \
              --quiet 2>/dev/null || echo "000")
            echo "Judge0 health check attempt $i: HTTP $STATUS"
            if [ "$STATUS" = "200" ]; then
              echo "health=ok" >> $GITHUB_OUTPUT
              exit 0
            fi
            sleep 15
          done
          echo "health=fail" >> $GITHUB_OUTPUT
          exit 1

      - name: Rollback on health failure
        if: failure() && steps.health.outputs.health == 'fail'
        run: |
          echo "::error::Judge0 /system_info health check failed. Rolling back to version ${{ steps.prev.outputs.prev_version }}"
          gcloud compute ssh ${{ secrets.JUDGE0_VM_NAME }} \
            --zone=${{ secrets.JUDGE0_VM_ZONE }} \
            --ssh-key-file=~/.ssh/id_ed25519 \
            --command="set -e; cd ~/judge0 && cp .env.rollback .env && docker compose pull && docker compose up -d"
          echo "::error::Rollback complete. Judge0 deploy FAILED — investigate before retrying."
          exit 1
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → GCP API | GCP_SA_KEY grants compute + storage admin on the two VMs |
| GitHub Actions logs → public | If secrets leak into stdout, they become public repo artifacts |
| Runner → VM via SSH | APP_VM_SSH_KEY + JUDGE0_VM_SSH_KEY grant root-adjacent access (docker socket) |
| Internet → Judge0 | Remains blocked; health probes via SSH onto VM localhost |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-02-S | Spoofing | Tag push triggering deploy | mitigate | Branch protection on `main` + tag pushes require collaborator access; GitHub Actions only runs from trusted refs |
| T-43-02-T | Tampering | workflow file edits | mitigate | Workflow changes go through PR review; pr-checks.yml gates merges |
| T-43-02-R | Repudiation | deploy action | accept | GitHub Actions logs retained 90 days; workflow run URL captured in deploy commit |
| T-43-02-I | Info Disclosure | secrets in logs | mitigate | GitHub Actions auto-masks secret values; explicit rule: no `set -x` in run blocks; no `echo ${secret}` anywhere |
| T-43-02-D | DoS (bad deploy) | production app or Judge0 | mitigate | Health check post-deploy; auto-rollback on failure; concurrency group serializes deploys (cancel-in-progress: false) |
| T-43-02-E | Elevation (leaked SA key) | GCP_SA_KEY | mitigate | Scope SA to compute.instanceAdmin on the two VMs + storage.admin on GCR bucket only; rotation runbook in Plan 43-04; future: workload identity federation (v1.5) |
</threat_model>
