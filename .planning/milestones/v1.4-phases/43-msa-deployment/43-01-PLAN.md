   ```hcl
   resource "google_compute_disk" "judge0_data" {
     name = "${var.judge0_vm_name}-data"
     type = "pd-standard"
     size = var.judge0_data_disk_size_gb
     zone = var.zone
   }

   resource "google_compute_instance" "judge0" {
     name         = var.judge0_vm_name
     machine_type = var.judge0_machine_type
     zone         = var.zone
     tags         = ["judge0", "internal-only"]

     boot_disk {
       initialize_params {
         image = var.judge0_boot_image
         size  = 20
       }
     }

     attached_disk {
       source      = google_compute_disk.judge0_data.id
       device_name = "judge0-data"
     }

     network_interface {
       network    = data.google_compute_network.primary.name
       subnetwork = data.google_compute_subnetwork.primary.name
       # NO access_config block → no public IP (D-03 internal-only)
     }

     service_account {
       email  = google_service_account.judge0_metrics.email
       scopes = ["cloud-platform"]  # narrowed by IAM role binding in service-accounts.tf
     }

     metadata = {
       enable-oslogin = "TRUE"
     }

     # Persistent disk survives VM restarts per D-04
     allow_stopping_for_update = true
   }
   ```

   ```hcl
   # Imported resource — do NOT destroy/recreate. Matches existing GCE VM.
   # Attributes below must mirror current live config. Task 3 runs terraform import.
   resource "google_compute_instance" "app" {
     name         = var.app_vm_name
     machine_type = var.app_vm_machine_type
     zone         = var.zone

       # initialize_params populated from live VM after import via `terraform show`

     network_interface {
       network    = data.google_compute_network.primary.name
       subnetwork = data.google_compute_subnetwork.primary.name

         attached_disk,
   ```

   ```hcl
   resource "google_compute_firewall" "judge0_from_app" {
     name    = "allow-judge0-from-app"
     network = data.google_compute_network.primary.name

     allow {
       protocol = "tcp"
       ports    = ["2358"]
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer laptop → GCP API | Terraform apply credentials (SA key or ADC) grant compute + IAM admin |
| Terraform state → GCS bucket | State file contains VM IPs, SA emails, tfvars may include project secrets |
| App VM → Judge0 VM | Only trusted internal traffic permitted (port 2358) |
| Internet → Judge0 VM | MUST be blocked (no public IP, no public firewall rule) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-01-S | Spoofing | Terraform state access | mitigate | GCS bucket uniform-bucket-level-access + IAM restricted to trainer GCP user + CI SA only (documented in README) |
| T-43-01-T | Tampering | Terraform state file | mitigate | GCS versioning enabled at bucket creation (README bootstrap step) — rollback to prior state available |
| T-43-01-I | Info Disclosure | tfvars with project_id | mitigate | `.gitignore` excludes `*.tfvars`; only `*.tfvars.example` committed |
| T-43-01-D | DoS (public Judge0) | Judge0 VM network interface | mitigate | No `access_config` block on Judge0 VM network_interface → no public IP provisioned |
| T-43-01-E | Elevation | Judge0 VM service account | mitigate | SA granted ONLY `roles/logging.logWriter`; no compute/storage/iam admin roles |
| T-43-01-F | Firewall misconfig | google_compute_firewall.judge0_from_app | mitigate | `source_ranges` bound to single /32 of app VM internal IP; no 0.0.0.0/0; comment tags rule with "internal-only per D-03" for grep-verification |
| T-43-01-R | Repudiation (who ran apply) | Terraform operations | accept | Solo developer; GCS state versioning + git history of .tf files is sufficient audit trail for v1.4 |
