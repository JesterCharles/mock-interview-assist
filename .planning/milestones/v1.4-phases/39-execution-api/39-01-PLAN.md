<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Next.js API | Untrusted body (challengeId, language, code), untrusted session |
| Next.js API → Judge0 | Server-to-server inside internal network (Phase 38 binding) |
| Next.js API → Private GitHub repo | Server-only token (`GITHUB_CODING_PRIVATE_TOKEN`) |
| Next.js API → Supabase DB | Service-role connection via Prisma |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-01 | Spoofing | POST /api/coding/submit | mitigate | `getCallerIdentity()` returns Supabase-validated user; associate kind requires matching `authUserId` FK. Anonymous → 401. |
| T-39-02 | Information Disclosure | Hidden test inputs in response | mitigate | `loadHiddenTests` called server-only; response shape is `{ attemptId }` only. Test assertion: response body never contains hidden stdin/expectedStdout strings. |
| T-39-03 | Denial of Service | Rate limit bypass | mitigate | `checkCodingSubmitRateLimit` atomic counter (file-backed, upgrade path to Redis in v1.5). 30/hour, 200/day per userKey. Retry-After header prevents retry-storm confusion. |
| T-39-04 | Tampering | Unsupported language crashing Judge0 | mitigate | Server-side allowlist check against `Object.keys(JUDGE0_LANGUAGE_MAP)` AND against `challenge.languages`. Rejected before Judge0 call. |
| T-39-05 | Elevation of Privilege | Associate submitting to another cohort's challenge | mitigate | D-02 authz: associate + non-null challenge.cohortId → must match caller's cohortId. Fetched fresh from DB each request. |
| T-39-06 | Denial of Service | Judge0 unavailable degrading app | mitigate | Timeout on judge0Client.submitBatch (Phase 38 D-12: 30-sec read). Catch → 503 JUDGE0_UNAVAILABLE. CodingAttempt rolled back on Judge0 failure so user can retry. |
| T-39-07 | Tampering | Malicious oversized code payload | mitigate | Zod schema caps `code: z.string().min(1).max(100_000)` (~100KB). Rejects at validation layer before any DB or Judge0 call. |
| T-39-08 | Repudiation | Who submitted what | accept | `CodingAttempt.associateId` + `submittedAt` + `submittedCode` persisted. Audit trail is implicit. No separate audit log needed for v1.4. |
</threat_model>
