<task type="auto">
  <name>Task 2: Wire prisma migrate deploy into Dockerfile</name>
  <files>Dockerfile</files>
  <action>
Add `prisma migrate deploy` to the Docker build pipeline so prod stays in schema sync per D-04.

The current Dockerfile (lines 38-45) runs `npx prisma generate` then `npm run build`. Migration deploy must run AT RUNTIME (not build time) because the build environment has no DB access — the build only generates the client.

Modify the Dockerfile runner stage (after USER nextjs, before CMD):

1. Change the `CMD` line to run migrations then start the server. Replace:
   ```
   CMD ["node", "server.js"]
   ```
   with:
   ```
   CMD ["sh", "-c", "npx prisma migrate deploy && node server.js"]
   ```

2. To make `npx prisma migrate deploy` available in the runner stage, copy the prisma CLI and migrations folder from the builder stage. After the existing COPY lines (~line 70), add:
   ```dockerfile
   COPY --from=builder --chown=nextjs:nodejs /app/node_modules/prisma ./node_modules/prisma
   COPY --from=builder --chown=nextjs:nodejs /app/node_modules/@prisma ./node_modules/@prisma
   COPY --from=builder --chown=nextjs:nodejs /app/prisma ./prisma
   ```

3. Verify the build succeeds locally:
   ```bash
   docker build -t nlm-test .
   ```
   Build should complete without error. Image size increase should be modest (~30-50MB for prisma CLI).

DO NOT run `prisma migrate deploy` at build time — the build runs in an isolated env with no DB credentials, and per PITFALLS.md, building with DB access creates secrets-in-image risk (per Dockerfile comment at line 84: "Secrets are NOT baked into the image").

DO NOT use `prisma db push` — that bypasses migrations and breaks the migration history. Always use `migrate deploy` per D-04.
  </action>
  <verify>
    <automated>grep -q "prisma migrate deploy" Dockerfile && docker build -t nlm-test . 2>&1 | tail -5</automated>
  </verify>
  <done>
Dockerfile CMD runs `prisma migrate deploy && node server.js`. Prisma CLI and migrations folder copied to runner stage. Local docker build succeeds. Image starts and applies migrations on first run.
  </done>
