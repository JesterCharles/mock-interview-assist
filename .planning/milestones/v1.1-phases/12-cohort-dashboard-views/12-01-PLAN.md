<objective>
Extend `/api/trainer` GET endpoint with two OPTIONAL query params, keeping the default response shape EXACTLY as v1.0:

- Default (no query params): `RosterAssociate[]` — UNCHANGED from v1.0. Existing `/trainer` (`src/app/trainer/page.tsx:33-38`) and `/dashboard` typeahead (`src/app/dashboard/page.tsx:79-84`) continue to work with zero changes.
- `?cohortId=X`: Same `RosterAssociate[]` shape, filtered to that cohort.

Per Codex review finding #1: the original plan to unconditionally change `/api/trainer` from `RosterAssociate[]` to `{associates, summary}` would break both current consumers during rollout. This plan preserves the contract and only introduces the wrapped shape behind an explicit opt-in flag that new cohort-dashboard code sets deliberately.

Purpose: Backward-compatible server-side filtering and aggregation for cohort dashboard (COHORT-03, COHORT-04) without atomic-release risk.
Output: Updated roster API + extended trainer types.
</objective>
