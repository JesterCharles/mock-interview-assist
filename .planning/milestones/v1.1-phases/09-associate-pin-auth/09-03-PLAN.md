<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/associate/pin/verify | PIN + slug from untrusted client |
| server component → DB | Associate slug → id resolution + pinGeneratedAt version comparison |
| trainer UI → /api/associate/pin/generate | Relies on trainer session cookie |
| server component → client boundary | Identity passed server→client as props, never trusted from client read |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-12 | Information Disclosure | /associate/[slug] cross-associate viewing | mitigate | Server component checks associateIdentity.associateId === targetId; mismatch returns 403 before data fetch (D-21) |
| T-09-13 | Elevation of Privilege | Client-side-only guard bypass | mitigate | Guard runs in server component (not client), cannot be bypassed by DOM manipulation |
| T-09-14 | Information Disclosure | PIN shown on trainer screen | accept | One-time display with "will not be shown again" warning + revocation note; user-mediated delivery; consistent with D-04 / D-19 |
| T-09-15 | Spoofing | Client-submitted slug to verify endpoint | mitigate | Verify endpoint (Plan 09-01) uses slug only to look up pinHash; success sets cookie with the server-resolved associateId + pinGeneratedAt, not client-supplied identity |
| T-09-16 | Denial of Service | Rapid PIN submissions | mitigate | Form disables during in-flight; 429 surfaced as visible error; server-side fingerprint rate limit (Plan 09-01) |
| T-09-20 | Spoofing | /associate/[slug]/interview identity forgery | mitigate | Identity resolved server-side via getAssociateIdentity (version-checked); passed to client as props; client cannot self-elevate (Codex #2 + #4) |
| T-09-21 | Elevation of Privilege | Stale cookie reaching authenticated interview entry | mitigate | getAssociateIdentity returns null for stale ver → redirect to login (Plan 09-02 version check) |
</threat_model>
