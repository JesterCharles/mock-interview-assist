# Milestone v1.3: UX Unification & Polish

**Status:** SHIPPED 2026-04-18
**Phases:** 26-35 (11 phases incl. decimal 28.1)
**Total Plans:** 18

## Overview

Unified all application surfaces onto the two-level app shell (topbar + sidebar), enriched the associate experience with curriculum visibility and richer data visualization (skill cards, focus area hero, radar overlay), introduced a Profile model migrating first-login detection off `user_metadata`, and closed out dark-mode coverage across the full surface area. Included a sidebar-primary navigation overhaul, accordion sign-in redesign, and a post-audit gap-closure wave (Phases 33-35) that wired the trainer first-login gate, replaced synthetic radar "Est. prior" with real per-skill historical snapshots, and reconciled the associate Settings accordion.

## Milestone Summary

**Decimal Phases:**

- Phase 28.1: User Profile (inserted after Phase 28 to introduce Prisma `Profile` model before it was consumed by later auth flows).

**Key Decisions:**

- `AppShell` role prop defaults to `'trainer'` — prevents trainer layout regression during shell refactor (Phase 27).
- Cohort ID derived from auth session (never query params) — prevents unauthorized curriculum data exposure (Phase 30).
- Build chart components with CSS vars from day one — avoids dark-mode retrofit in Phase 31 (Phase 26+).
- Cap gap history at 20 sessions for trend charts — perf guard (Phase 29).
- Profile model keyed on `authUserId` only, no FK to Associate — trainers get profiles too (Phase 28.1).
- Lazy backfill over batch migration for `Profile.passwordSetAt` — handles user_metadata→Profile migration organically (Phase 28.1).
- SkillRadar Before/Now overlay replaces per-skill LineChart as canonical trajectory visual — VIZ-03 cut (Phase 34).
- `GapScore.prevWeightedScore` is nullable with no backfill — populates on next upsert; radar hides Before polygon until data exists (Phase 34, D-10).
- Settings accordion uses CSS max-height transition (0 → 300px) — avoids height:auto animation problem (Phase 32).

**Issues Resolved:**

- SHELL-01 scope drift (3-item nav vs 2) — reconciled in REQUIREMENTS.md + DESIGN.md App Shell section (Phase 35).
- SIGNIN-02 trainer first-login gate — closed via exchange route reorder + SignInTabs client gate (Phase 33).
- VIZ-03/-05/-06 named-vs-default import regression — closed via Phase 34 (SkillTrendChart intentionally removed, SkillRadar renders real snapshots, filter sync locked by integration test).
- `trainer.css` missing `[data-theme='dark']` scope — fixed in Phase 31 late touch-up.
- `ProfileTabs.tsx` hardcoded badge hex — fixed (tokens in Phase 31 late touch-up).
- Phase 32 Plan 01 worktree-merge gap (SHELL-32-01/-02/-03) — all merged; TopBar has no NAV_ITEMS/resolveGroups; Batch Upload in Actions; settings accordions exported + consumed.

**Issues Deferred:**

- Nyquist validation hygiene (VALID-01) — only Phase 26 has a `-VALIDATION.md` (draft). Deferred as tech debt; not a milestone blocker.
- Phase-level VERIFICATION.md normalization for Phases 26 and 35 (evidence exists in SUMMARY.md + grep blocks).
- Historical Phase 28/29/31/32 VERIFICATION.md artifacts are pre-gap-closure; not updated to post-closure state (audit cross-references Phase 33-35 closure).

**Technical Debt Incurred:**

- `prisma migrate deploy` for `prevWeightedScore` column deferred to normal deploy flow (not run in Phase 34).
- Most 999.x backlog items (staging/prod split, trainer default cohort) carried forward to v1.4 planning.
