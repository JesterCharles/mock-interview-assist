## Overview

v1.1 evolves Next Level Mock from a single-associate Readiness Loop (v1.0) into a cohort-first readiness engine. Trainers can group associates into cohorts, define weekly curriculum schedules, and filter dashboards / setup wizards by what's been taught. Automated interviews are fully wired into the gap / readiness pipeline through a new authenticated entry — anonymous automated sessions remain supported but can no longer forge associate linkage. The entire UI converges onto a single DESIGN.md token system (warm parchment + Clash Display + DM Sans + burnt orange accent), with the legacy `--nlm-*` dark-theme stack deleted.

## Milestone Summary

**Key Decisions:**

- **PIN auth feature-flagged off for v1.1 ship** (`ENABLE_ASSOCIATE_AUTH=false`). Codex P1 flagged limiter as trusting client fingerprint; defense-in-depth + flag-gate chosen over mid-ship rearchitecture. v1.2 will harden further and flip the flag.
- **Legacy `--nlm-*` deletion** (Codex #8 override). Two parallel token systems is retrofit debt a solo dev cannot amortize; Phase 15 scope-expansion added DESIGN-03 to migrate all remaining surfaces and atomically delete the legacy stack.
- **Idempotent baseline migration**. Hand-wrote `0000_baseline/migration.sql` from git snapshot; Codex follow-up rewrote with `IF NOT EXISTS` + `duplicate_object` FK guards so migrations are safe against both fresh and pre-existing DBs.
- **Composite rate limiter for PIN verification** — `(server-observed IP, fingerprint)` + IP-only bucket. `NLM_TRUSTED_PROXY=true` opts into `x-forwarded-for`.
- **Split completion endpoints (Codex #3)**. Anonymous strips identity unconditionally; authenticated derives identity from cookie. Forged-linkage attack surface eliminated by construction.
- **`Session.readinessRecomputeStatus` + sweep endpoint (Codex #5)**. `pending` → `done`/`failed` marker makes failed recomputes repairable without backfill. `runReadinessPipeline` returns `boolean` so sweep doesn't false-close siblings on throw (P2 fix).
- **Opt-in dashboard summary shape (Codex #1)**. `GET /api/trainer` default unchanged (`RosterAssociate[]`); wrapped shape opt-in via `?includeSummary=true`. No v1.0 consumer breaks.
- **Exact skillSlug match, not substring (Codex #9)**. `Set.has(firstSegment)`; `@@unique([cohortId, weekNumber])` at DB level.
- **`<400ms` wizard perf is target, not gate (Codex #7)**. Real bottleneck is recursive GitHub question-bank discovery. Cached manifest deferred to v1.2.
- **Dedicated `ASSOCIATE_SESSION_SECRET`, not `APP_PASSWORD` (Codex #4)**. Decouples associate auth from trainer password rotation; HMAC-grade secret + `pinGeneratedAt` token version gives DB-backed revocation without a session table.

**Issues Resolved:**

- Automated interviews now link to authenticated associates; cannot be forged
- Anonymous public completion path preserved + hardened
- Readiness recompute is now repairable via sweep endpoint
- Cohort filter/summary added without breaking v1.0 API consumers
- Curriculum-driven question selection works via canonical skillSlug
- Single unified design system; legacy dark-theme token stack deleted
- Split logins consolidated into `/signin` tabs

**Issues Deferred (to v1.2):**
