## Cross-Phase Integration: 19/22 Wired

### Medium Issues (3)

**INT-01: readinessScore always null** (DASH-01, DASH-05)
Both `/api/trainer` and `/api/trainer/[slug]` hardcode `readinessScore: null`. The Associate model has no numeric score column. ReadinessDisplay component's numeric score rendering is dead code. The readiness STATUS badge works correctly.

**INT-02: Public interviews skip gap/readiness pipeline** (GAP-01, READY-01)
`/api/public/interview/complete` calls `persistSessionToDb` only — no `saveGapScores` or `updateAssociateReadiness`. Associates doing only public interviews never get gap scores. Gap computation triggers on next trainer-led session for the same associate.

**INT-03: Skill name mismatch for nested repos** (GAP-05)
`techMap` stores `tech.name` minus `.md`, but `applyGapScores` extracts directory name from `tech.path`. Mismatch occurs only with nested-directory question bank repos (e.g., `javascript/question-bank-v1.md`). Flat-structure repos work correctly.

### Low Issues (2)

**INT-04: getGapScores orphaned** (GAP-03)
`gapPersistence.ts` exports `getGapScores` with 3-session gate, but nothing imports it. Phase 07 created a separate endpoint querying Prisma directly. Dead code — gate still enforced elsewhere.

**INT-05: No settings UI** (READY-03)
`/api/settings` GET/PUT is tested (16 tests) and functional but has no frontend. Trainers cannot change threshold through the app. API-only for now.
